# PT-2 每日签到

## 状态 I：未签到态

### 模块：顶部导航

- 返回操作：点击后返回 CC-3 主导航-我的
- 页面标题：「每日签到」

### 模块：签到主体区

- **状态图标**：代表"未签到"语义的日历图标
- **状态标题**：「今日还未签到」
- **积分说明**：「每日签到可获得 N 积分」，N 由积分配置表中的 `daily_checkin_points` 字段决定，前端实时读取展示
- **签到按钮**：主操作按钮，文案「立即签到」，全宽，可点击
  - 点击触发：调用签到接口 → 接口返回成功后切换至状态 III（签到成功反馈）
  - 点击触发：按钮立即切换为不可点击态（防止重复提交，无需等待接口返回）
- **连续签到天数**：「已连续签到 · N 天」，N 为当前用户连续签到计数，从接口获取；若从未签到则显示「已连续签到 · 0 天」

### 模块：业务约束

- 每自然日（00:00–23:59）仅允许签到一次；已签到用户直接进入状态 II
- 连续签到天数计算规则：上一自然日未签到则断链，计数归零（从次日签到起重新计算）
- Phase 1 不设连续签到奖励；N 天连续计数仅用于展示，无额外积分逻辑

---

## 状态 II：已签到态

### 模块：签到主体区

- **状态图标**：代表"已完成"语义的勾选图标
- **状态标题**：「今日已签到」
- **积分说明**：「今日已获得 N 积分，明天再来」，N 为本次签到实际到账积分数
- **签到按钮**：不可点击态，文案「已签到 ✓」，吞掉所有点击事件，无任何响应
- **连续签到天数**：含今日的最新连续天数（比签到前 +1）

---

## 状态 III：签到成功反馈（过渡态）

### 模块：积分到账通知

- **通知形式**：全局悬浮提示条，覆盖于主体区上方
- **通知文案**：「今日签到成功，获得 N 积分！」，N 为本次签到实际到账积分数
- **存续时长**：2 秒后自动消失，消失后通知区域归还给页面内容
- **交互属性**：只读展示，无点击跳转

### 模块：主体区切换

- 积分到账通知出现的同时，主体区同步切换为状态 II（已签到态）的视觉呈现
- 过渡为即时切换，无淡出/淡入动效要求
