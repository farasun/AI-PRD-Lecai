# CC-3 主导航-我的（积分卡片区块）

> **文件说明**：本 Spec 仅覆盖 CC-3「我的」页在 v1.1 新增的积分卡片区块。CC-3 原有模块（用户信息、社团入口、作品入口等）已有独立 Spec，不在本文件范围内。

## 状态 I：积分卡片 - 未签到态

### 模块：积分卡片

- **卡片位置**：位于用户信息区下方、功能菜单列表区上方，整体为独立卡片容器
- **模块标题**：「我的积分」，右侧带跳转图标
  - 点击标题行区域：跳转 PT-1 我的积分
- **积分余额**：展示用户当前积分总余额（整数），数据来源与 PT-1 余额展示一致，进入页面时加载
  - 单位文字「积分」紧随余额数字
- **今日签到按钮**：主操作按钮，文案「今日签到 +N 积分」
  - N 为今日签到可得积分数，从积分配置表中的 `daily_checkin_points` 字段读取
  - 点击：跳转 PT-2 每日签到
- **卡片点击区域说明**：
  - 标题行（「我的积分」文字 + 跳转图标区域）：跳转 PT-1 我的积分
  - 余额行：跳转 PT-1 我的积分
  - 签到按钮：跳转 PT-2 每日签到（签到按钮不触发 PT-1 跳转）

### 模块：业务约束

- 积分余额加载失败时：数字区域展示加载占位，签到按钮正常可点击（两者数据独立）
- 积分为 0 时：数字展示「0」，按钮状态不变

---

## 状态 II：积分卡片 - 已签到态

### 模块：积分卡片（差异描述）

- 今日签到按钮切换为不可点击态，文案变为「今日已签到 ✓」
- 余额行、标题行跳转逻辑不变
- 判断逻辑：进入 CC-3 时，接口返回当日签到状态；若状态为"已签到"则直接呈现已签到态
- 从 PT-2 签到成功返回 CC-3 后，按钮应同步更新为已签到态（无需刷新页面，由本地状态管理处理）

---

## 状态 III：积分卡片 - 数据加载中

### 模块：积分卡片（差异描述）

- 积分余额数字区域：展示加载占位符（骨架屏，三个横线）
- 签到按钮：不可点击态（防止在余额数据未就绪时误操作），按钮文案保持「今日签到」，不展示积分数（因数据未就绪）
- 加载完成后：切换至状态 I 或状态 II（根据当日签到状态决定）
