| Flow B | 上传作品得积分 | 普通成员 | R2 / R3 | ✅ 已产出 |

## 二、各流程拆解

### Flow A · 每日签到得积分（普通成员）

**入口页**：CC-3 主导航-我的（积分卡片内签到按钮）

```
CC-3（积分卡片 · 签到按钮）
    → 点击签到
    → [系统] 判断今日是否已签到
        ├─ 未签到 → PT-2 每日签到（触发签到动效 + R1 积分到账 Toast）
        └─ 已签到 → PT-2（展示已签到态，按钮不可点击）
```

**涉及页面**：CC-3、PT-2
**Toast 触发页**：PT-2

---

### Flow B · 上传作品得积分（普通成员）

**入口页**：CC-7 社团提交作品表单（从社团活动详情进入）

```
CC-7（提交作品表单）
    → 用户填写并提交
    → [系统] 进入审核队列
        ├─ 审核通过 → CC-7（当前页或返回后）触发 R2 积分到账 Toast
        │              └─ 若为首次发布 → 同时触发 R3 新手奖励 Toast（叠加发放）
        └─ 审核拒绝 → 不触发积分
```

**涉及页面**：CC-7
**Toast 触发页**：CC-7

---

### Flow C · 查看积分明细（普通成员）

**入口页**：CC-3 主导航-我的（积分卡片区域）

```
CC-3（积分卡片）
    → 点击卡片主体
    → PT-1 我的积分（余额 + 流水记录）
        → 点击签到入口
        → PT-2 每日签到
```

**涉及页面**：CC-3、PT-1、PT-2

---

### Flow D · 加入 / 创建社团得积分（普通成员 + 社团主）

**入口页 D1（加入社团）**：CC-15 社团成员申请加入弹窗

```
CC-15（申请加入弹窗）
    → 用户提交申请
    → [系统] 判断社团是否需审核
        ├─ 直接加入 / 审核通过 → CC-15（当前页或通知）触发 R6 积分到账 Toast
        │                         └─ 若为首次加入任何社团 → 同时触发 R7 新手奖励 Toast（叠加）
        └─ 审核拒绝 → 不触发积分
```

**入口页 D2（创建社团）**：CC-9 申请创建社团

```
CC-9（申请创建社团表单）
    → 用户提交申请
    → [系统] 进入平台审核
        ├─ 审核通过 → CC-9（通知态）触发 R8 积分到账 Toast
        └─ 审核拒绝 → 不触发积分
```

**涉及页面**：CC-15、CC-9
**Toast 触发页**：CC-15（R6 / R7）、CC-9（R8）

---

### Flow E · 社团主阶梯达标得积分（社团主）

**入口页**：CC-4 社团主页（被动触发，无明确入口操作）

```
[系统后台] 实时监测社团数据变化
    ├─ 团员数达到新阶梯（R10）
    │       → CC-4 社团主页触发阶梯达标积分到账 Toast
    └─ 作品数达到新阶梯（R11）
            → CC-4 社团主页触发阶梯达标积分到账 Toast
```

**说明**：Flow E 为被动触发流程，社团主在社团主页时系统实时推送，无用户主动操作节点。
**涉及页面**：CC-4
**Toast 触发页**：CC-4（R10 / R11）

## 四、设计约束（继承自 li-yue-review.md）

- **Toast 文案**：必须口语化，禁用互联网黑话（参见 `li-yue-review.md` 第二节各条意见）
- **R10 / R11 推送时机**：实时触发，不可延迟
- **Toast 展示形式**：全局浮层，持续 2 秒自动消失，无点击跳转（参见 `doc/points-system-v1.1.md` 第一节第 3 条）
