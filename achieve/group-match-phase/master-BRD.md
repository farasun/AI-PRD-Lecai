# 乐才赛事A模式 - 宏观业务总规 (Master BRD)

## 1. 业务愿景 (Vision)
以官方大型赛事为指挥棒，通过“社团参赛”这种集体行为，大规模带动用户在平台创建社团、建立真实社交关系。用户不再是一个人刷视频，而是作为某个集体的代表出战。

## 2. 核心架构设计 (S2B2C)
*   **S (Platform)**：官方负责搭建大赛事，配置公平系数，进行反刷票监管。
*   **B (Club Leader)**：团长负责“选派”优秀成员作品，代表社团出赛，行使“推荐权”。
*   **C (Member)**：成员投稿并为自己的集体“加油、喝彩”，通过集体荣誉感实现高留存。

## 3. 全局迭代蓝图 (Phasing Roadmap)

### v1.4a (核心流转版) - 交付重心：业务闭环
*   **主要功能**：
    - 后台端：A模式赛事创建开关。
    - 团长端：赛事报名功能，【选送确认】动作。
    - 成员端：【代表单位投递】选择器，作品带社团标签显示。
    - 榜单：基础人气榜（社团作品点赞之和）。

### v1.4b (公平与风控增强版) - 交付重心：算法与合规
*   **主要功能**：
    - 算法：引入【公平因子】计算，(总点赞/参赛数)*系数。
    - 风控：反刷票规则与非法积分自动扣除逻辑。
    - 法律建议：版权说明、赛事规则公示等文案规范落地。

### v1.4c (运营体系优化) - 交付重心：规模化运营
*   **主要功能**：
    - **【组别划分】**：支持根据社团规模自动分流竞赛组（如“新锐组”、“资深组”）。
    - 公告：官方【荣誉快报】自动生成功能。

## 4. 关键成功衡量指标 (Success Metrics)
*   每场官方赛事带来的“新创建社团”转化率。
*   社团内成员的平均赛事投稿参与率（Participation Rate）。
