# 乐才赛事A模式 - v1.4b (核心闭环版) 当前迭代草案

## 1. 核心业务价值
*   **本期目标**：跑通“社团报名 -> 成员投稿 -> 团长确认 -> 基础人气排行榜”的完整闭环流程。
*   **最小可行性流转 (MVP)**：确保赛事能通过后台配置产生，并有明确的社团标识在移动端分发。

## 2. 极简改动圈定 (Scope)

### **A. 后台端 (S端)**
*   赛事后台新增模式开关，支持标记为“A模式（官方主导）”。
*   配置各社团成功报名后的初始属性。

### **B. 团长端 (B端)**
*   在社团管理页面新增【赛事报到】入口，显示当前的官方赛事。
*   收到成员投稿后的【确认申请】动作（若不确认，则不挂件为该社团代表作品、不计积分）。

### **C. 成员端 (C端)**
*   投稿选择框：仅可选择已成功报到的社团。
*   作品详情：作品下方必须有“代表 [XX社团] 选送”字样图标。

## 3. 基础业务规则 (High Priority)
*   **代表单位参报规则**：用户在多社团场景下，单个作品只能选择一个社团。
*   **社团人气规则 (Basic)**：当前阶段暂按“作品点赞之和”排序，后台需预留成员总数接口以备后续算法升级。
*   **入选门槛**：当前暂不做社团人数强校验（2人亦可），作为业务观察项。

## 4. 文案规范 (银龄语库应用)
*   **按钮**：“确认选送推介”（原审核按钮）。
*   **页签**：“社团风采录”（原排行榜页签）。
*   **提示**：“感谢为您代表的集体喝彩助力”。

## 5. 专家红线约束 (@elder-expert-liyue)
*   **恶意刷票警告**：在投票确认弹窗中必须明示“严禁刷票，系统将自动监控异常票选并扣除社团积分”的行政警告。
*   **版权界定**：投稿时必须勾选《集体代表作品版权说明》，明确作品展示权与社团主体的强绑定关系。
