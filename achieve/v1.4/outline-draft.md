# 乐才赛事A模式 - 初步愿景大纲 (v1.4)

## 1. 核心愿景与商业目的
*   **愿景**：打造平台级“大赛事”标杆，以官方流量引导用户“创建社团”并“代表社团参赛”。
*   **商业目的**：增加平台社团基数，提升核心活跃度，形成“团员参赛 -> 为团争光 -> 促进拉票活跃”的正向业务循环。

## 2. 核心业务流程 (MVP)
1.  **平台建赛**：官方在管理后台开启 A 模式赛事（如“京津冀大赛”）。
2.  **团长报赛**：团长代表其所属社团在移动端/小程序端申请参加该官方赛事。
3.  **成员投递**：社团成员在投稿时，仅可选择已成功报名的社团进行“挂载”投递。
4.  **积分累积**：社团在赛事中的人气榜总分为“该社团下所有参赛成员作品点赞数之和”。
5.  **结果公示**：线下评比完成后，官方在平台上进行获奖公告发布。

## 3. 详细业务规则
*   **参赛限定 (Strict Limit)**：仅允许已报名的社团参赛，彻底取消个人报名通道。
*   **身份标识 (Branding)**：参赛作品在全局展示（推荐流、排行板）中必须带有明确的“所属社团”徽章。
*   **榜单逻辑 (Ranking Logic)**：
    - 个人人气榜：展示单体作品热度。
    - 社团总榜：展示社团内所有作品累计热度，引导团长组织团员拉票。
*   **反向导流 (Traffic)**：社团详情页应能直接看到该社团正在参加的赛事活动，并展示相应的作品相册。

## 4. 风险与优化关注点 (TBD)
*   **活跃门槛 (Deferred)**：当前不强制校验社团人数（如最低2人），作为后续版本优化的预留监控项，以防产生大量仅为了比赛而创建的“一人社团”。
*   **一人多团**：允许多社团身份，但在投稿时必须进行“单一代表权”选择。

## 5. 专家审核重点 (Li Yue Focus)
*   如何防止大场面赛事中的恶意刷票行为？
*   对于“银发社团”中团员较少、活跃程度一般的边缘化社团，A 模式是否会造成严重的头部垄断而导致小团积极性丧失？
*   作品版权与社团作为主体的归属权纠纷风险评估。
