# 专家前置审阅 (Step 0) - v1.3.2a 赛事闭环与平台代制模式

**审阅人**：李越（银发经济产品专家）
**审阅时间**：2026-04-03
**审阅背景**：基于 v1.2 卡哇伊试点教训与 v1.3.2 拆分策略，验证“平台代建+C端闭环”的 MVP 模式。

---

## 1. 核心业务价值评估 (S2B2C 视角)
本次 v1.3.2a 的核心战略是 **“低成本、重荣誉、强卡点”**。
我们在卡哇伊 v1.2 试点中发现，中老年用户对“入社”本身的抵触不高，但对“入社后再找报名入口”的多步操作感知极差。v1.3.2a 的“插件化”理念完美解决了这个问题：
- **赋能 B 端（团长）**：虽然赛事由平台代建，但所有荣誉和管理权挂载在团长名下，极大满足了其“领袖欲”。
- **转化 C 端（成员）**：通过“不入社不能投/不能投”的刚性拦截，将赛事参与直接转化为社团留存，这是最硬核的拉新指标。

---

## 2. 基于 v1.2 试点的经验修正与关键审阅意见

### 2.1 拦截交互：从“报错”进化为“引导” (CC-5-1 关联)
- **v1.2 痛点**：用户点击报名弹出“请先加入社团”，导致流失。
- **v1.3.2a 要求**：必须实现**“一键沉浸式入社”**。在赛事详情页，针对非成员显示的不是“报名”，而是“加入社团并参赛”。点击后，后台完成入社动作，前端直接刷新为报名/投稿态。**严禁跳转到 CC-4 再跳回来。**

### 2.2 荣誉外显：电子奖状是“社交核武器” ✅
- **李越建议**：中老年人参加比赛就是为了“显摆”。我们在 a 阶段虽然不给团长做自助工具，但**结赛后的荣誉喜报和电子奖状必须是高规格的、可长按保存的精美图片**。
- **UI 要求**：奖状底部必须明确标识【XX社团 颁发】，并带有团长名。这将极大刺激团长在微信群二次传播。

### 2.3 管理权插件化：给王阿姨最直接的“操作杆”
- **页面设计倾向**：参照 CC-4 社团主页插件化思路。在已有的活动详情页中，如果当前用户是该社团团长，且该活动是“赛事类”，则自动点亮如下功能：
    - **【作品审核】**：团长可以长按下架违规作品（比如张大妈发了一张模糊不清的照片）。
    - **【实时榜单预演】**：团长能实时看到当前的票数分布，方便其在群里“喊话”拉票。

---

## 3. S2B2C 业务逻辑卡位
- **S 端（平台）**：通过 Web 大后台承载所有“脏活累活”（配置时间、奖项等），确保 v1.3.2a 研发资源集中在 C 端互动。
- **B 端（团长）**：获得“特权感”。他是赛事的“总导演”，拥有“封禁违规作品”的权力（虽然戏台是平台搭的）。
- **C 端（参赛者）**：路径极简。入社 -> 投稿 -> 分享拉票 -> 拿电子奖状。

---

## 4. 结论与下一步指引
**结论：同意 v1.3.2a 的目录结构与功能边界。** 这种“先跑通 C 端体验，后台人工配置兜底”的策略是应对银发市场波动的最佳方案。

**下一步要求：**
1. **产品经理** 立即根据本意见细化 P0 级 PRD。
2. **设计方案** 需重点产出“电子奖状”的视觉样张，必须符合中老年审美（红金配色、庄重感）。
3. **技术团队** 重点攻克“一键沉浸式入社”的原子化能力。

**结论：放行（Approved）。** 建议立即推进 PRD 编写。
