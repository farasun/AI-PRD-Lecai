# 业务规则总纲 (v1.3.2a) - 赛事闭环与平台代制

## [现状规则摘要]
在以往的 v1.0 至 v1.2 版本中，系统主要提供了以“社团”为主体的极简照片上传与积分功能；赛事模块 (CH系列) 虽在字典中有编号，但并未真正落地业务闭环，且与“强社团属性”存在架构撕裂。
本次迭代彻底**废弃使用独立的大众赛事体系（CH 系列）**，将赛事完全降级为社团内部架构的一个超级组件（全盘使用 **CC 系列** 承载），实现 C 端的“参赛 - 投票 - 发奖”链路与“所属社团身份”的强绑定闭环。

---

## 一、 核心业务边界

1. **强社团绑定策略 (入社前置卡点)**
   - **拦截条件**：任何用户欲在官方赛事中执行**投稿、投票、报名**操作时，系统必须强制校验其是否为该赛事“指定社团”的正式成员。
   - **拦截动作 (CC-15 专属包装)**：若非成员，立刻阻断当前操作，并强制拉起 CC-15 入社弹窗。弹窗文案严禁使用“您无权限”等系统黑话，必须通过情绪价值包装，例如：“赛事专属战队组建中：快来入队为 XX 社团助威！”，实现以赛事拉新促活的核心业务目标。
   - **外链分享一键闭环 (防迷航)**：处理从外部直接分享进入的 CC-24（社团赛事专属主页）或 CC-6（社团作品详情展示页）时，引导入社（CC-15）的链条必须确保“一键闭环”。即：外来老人点击「加入战队」后，系统自动在后台完成“入社 + 鉴权授权”动作并**原地刷新/直接加载至原分享的终点页面**。切忌在入社成功后把用户丢回首页或大厅让他们重新寻找赛事。
   - **独立通道封锁**：系统不再开放脱离社团概念的纯个人常规报名入口。

2. **C端体验闭环设计 (社团内嵌化)**
   - **并列TAB架构**：社团赛事核心不以外跳独立页面（如传统的 CH-1）为主路径，而是直接集成至 **社团主页 (CC-4)** 中，形成「活动」与「赛事」并列的两个 TAB。
   - **沉浸式赛事专属区 (CC-4-2)**：在社团主页-赛事 TAB 中直观展示赛事信息卡、本社团成员的参赛作品及排行，使得赛事完全融入社团的生命周期中。
   - **大舞台品牌感知**：当用户进入赛事详情时，核心区域固定展示冠名方（特定社团/赞助商名称），建立官方权威背书。
   - **生命周期严控**：赛事严格受限于“起止时间”。进行期间开放点赞投票；到达截止时间后业务状态冻结，所有互动阀门关闭，列表转为公示模式。
   - **重荣誉、大字体反馈**：结赛时系统自动计算最终热度排行，为上榜作品页面挂载大字体的“电子奖杯/大奖状”，内嵌该社团长（小 B）的颁发背书和电子公章，满足极高的社交炫耀需求。

3. **B/S端分离 (平台代办机制) 与 B 端管理闭环**
   - 坚决剔除任何将复杂的“创赛图文表单”暴露给移动端团长的设计。赛事名称、规则明细、截票倒计时的初始化配置均由平台运营通过 Web 后台（CC-22）统一下发配置。
   - **新增赋能（CC-25 团长赛事管理规则台）**：填补管理侧空白，允许团长在移动端随时查询自己社团下属赛事的规则和进度。
   - **自动化状态同步**：团长管理台需实时拉取平台配置的五个关键时间节点（开始-投稿截止-投票截止-公示开启-结束），展示自动化阀门的当前状态。

---

## 二、 系统自动化流程控制 (生命周期阀门)

为了实现“名利双引擎”中的荣誉闭环，系统必须严格执行以下自动化阀门逻辑：

1. **投稿通道自动管控**
   - 触发时刻：到达平台配置的「作品提交截止日期」。
   - 动作：系统自动封死投稿入口（CC-7 及相关作品上传组件），对新产生的投稿行为予以拦截并提示“投稿已截止”。
2. **投票通道自动管控 (Anti-Fraud)**
   - 触发时刻：到达平台配置的「作品点赞（投票）截止日期」。
   - 动作：系统自动冻结所有作品详情页（CC-6）的点赞组件，锁定当前得票数据，不再支持二次点赞/撤销。
3. **自动化获奖标记**
   - 计算逻辑：投票通道关闭后，系统自动根据最终点赞（投票）数进行降序排列。
   - 执行动作：根据预设的奖项名额（由平台代录入），自动为对应作品挂载“电子奖杯/获奖勋章”标识。

---

## 三、 结果公示与荣誉闭环

1. **获奖公告模块 (Hot-Fix)**
   - **展示载体**：CC-24 赛事专属主页。
   - **自动激活**：当赛事流转至「公示期」状态后，系统需自动在页面顶层或专属 TAB 中推送「获奖公告」。
   - **内容构成**：中奖作品缩略图、对应中奖名次（如：人气冠军/优秀奖）、平台代录入的团长颁奖说明（增加权威感）。
2. **电子大奖状沉淀**
   - 系统定时结账并落定总榜单。不涉及真实物理奖品分配，但必须通过系统全自动生成包装精良、大字号的“电子奖杯图片”与荣誉喜报，里面必须内嵌清晰的团长颁发背书和公章印记。

---

## 四、 团长管理直通车
   - 在 C 端的社团赛事主页（CC-24）内，植入身份探测器。若当前用户为对应关联的社团长（班长），则直接激活其隐性权限视图，允许其“随手”删除违规内容或劣质作品。

5. **数字资产回收机制 (长期留存)**
   - 参赛者在赛事中提交的作品（图片/视频），系统底层将自动镜像并入所绑定的“专属社团”常规动态池或风采馆中。即使赛事落幕，这些参赛影像依然作为社团的永久数字资产持续留存展示，彻底解决“搞完活动就散伙、零资料沉淀”的痛点。
