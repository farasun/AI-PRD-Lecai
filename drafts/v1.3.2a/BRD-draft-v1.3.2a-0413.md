# 迭代切片：v1.3.2a 赛事增长验证与全网互动模式

## 1. 迭代目标
以最低门槛跑通“分享裂变 -> 互动拉新 -> 参赛转化”的增长模型。本期重点在于通过“非成员亦可点赞/评论”释放广告位 AD-A3 的传播潜力，同时通过“只有入社方可参赛”锁定转化。

## 2. 核心业务规则
### 2.1 参与权限分布
- **全网互动 (无拦截)**：点赞与评论对所有用户公开。对接 **广告 AD-A3 (作品拉票)**。
- **入社参赛 (强卡点)**：只有加入对应社团的成员方可进行“上传作品”和“报名”。对接 **广告 AD-A2 (邀请参赛)**。

### 2.2 排行与“轻量”闭环
- **排行规则**：沿用官方赛事的[点赞数+评论数]加权计算公式。
- **赛事结果图文公告 (MVP)**：结赛后，首页置顶展示一段由团长（平台代制）发布的图文公告，描述获奖情况并致谢。用于取代复杂的证书系统，提供情感闭环。

### 2.3 生命周期节点管理 (Manual Control)
- **管理模式**：本期不开发自动定时功能。所有节点由平台管理后台通过 5 个物理开关人工控制：
    1. **可见性开关**：控制活动在发现页/社团页的露出。
    2. **投稿开关**：控制作品上传通道。
    3. **点赞/评论开关**：控制互动拉票通道（关闭即固化排行）。
    4. **结果公示开关**：控制“结果公告”图文区块的展示。
    5. **活动下架开关**：控制活动彻底隐藏/进入历史区。
- **理由**：为团长赋予行政调优的“临时裁决权”（例如临时延长投票期），符合社团运营实情。

### 2.4 广告场景支持 (Advisement Interface)
- **定义来源**：`foundation/roadmap/advisement/iteration-v1a.md`
- **AD-A2 支持**：在邀请参赛分享落地页，按配置展示弹窗广告。
- **AD-A3 支持**：在作品点赞/评论详情页，按配置展示拉票弹窗广告。
- **AD-A5/AD-A7 支持**：在赛事主页及照片墙展示固定 Banner 广告。

## 3. 团长端限定
- 团长维持对代建赛事的审核/下架权限，以及提供结果公告素材的权力。

## 4. 本期剔除功能
- 颁奖页、公告中心（独立页面版）。
- 系统自动生成电子证书、荣誉大图。
- 团长端自助发赛入口（移至 B）。
