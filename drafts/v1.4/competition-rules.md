# 官方荣耀赛 V1.4 业务规则总纲

## 一、[现状规则摘要]
*   **赛事创建**：S 端由运营人员在 Web 后台手动创建赛事，配置名称、起止时间及奖项。
*   **作品提交**：用户直接上传图片/视频，目前无严格的身份绑定 and 前置拦截。
*   **排行榜**：基于点赞数生成简单的个人人气榜。

---

## 二、官方赛事核心逻辑

### 1. 准入、报名与管理链条 (Membership & Registration Flow)

1.  **赛事报名 (CH-1 -> CH-1-5)**：
    *   **入口**：团长进入赛事详情页 (`CH-1`)，系统识别身份后展示“立即报名”按钮；
    *   **表单内容**：进入 `CH-1-5`，系统自动锁定“代表社团名称”，团长仅需勾选内置寄语并确认联系电话。支持多场赛事并行报名。
2.  **官方审核 (S-End)**：
    *   平台管理员在 Web 后台 (`CC-24`) 对社团资格进行审核（通过/拒绝）；
    *   审核结果通过站内信实时触达团长端。
3.  **赛事管理中心 (CC-10 -> CC-10-3)**：
    *   **入口**：团长在“我的社团管理 (`CC-10`)”中通过“官方赛事”磁贴进入 `CC-10-3`；
    *   **功能**：`CC-10-3`（赛事记录中心）展示历史报名记录（含驳回原因）、当前已参加的赛事列表（点击跳转 `CH-1`），以及激活“分享邀请函”入口。
4.  **成员校验 (C-End)**：
    *   投稿前置拦截：用户投稿前强制执行实名及入社校验；若所属社团未报名该赛事，展示“社团未参赛”引导页并拦截进入 `CH-3`。
    *   作品归属绑定：投稿流程中强制选择代表社团，作品与社团 ID 永久绑定。

### 2. 情感传播与花絮板块 (Engagement & Highlights)

#### 2.1 配套活动机制（B 端管理）
*   **自动创建与标识**：审核通过后自动创建的配套活动，在社团内部活动列表 (`CC-12`) 中展示“官方赛事花絮”专属视觉标识。
*   **活动详情页 (CC-5) 差异化**：
    *   顶部显眼位展示“官方荣耀赛 · 专属花絮相册”标识及赛事跳转入口；
    *   操作规范：锁定活动名称、锁定活动封面，**隐藏/禁用删除按钮**；
    *   支持团长通过 `CC-12-1` 动态调整全员上传或仅限团长上传的权限。

#### 2.2 多维展示规则 (C 端视觉)
*   **作品详情页 (CH-2)**：显著位置同时展示“所属赛事”与“所属社团”的双重归属标签，增加社团荣誉感。
*   **赛事花絮列表 (CH-5)**：布局采用**瀑布流**列表形式（参考 `CC-6` 规范），聚合展示所有参赛社团的配套活动内容。
*   **花絮详情页 (CH-5-1)**：面包屑/导航区需同时展现：`[赛事名] > [社团名] > [活动名]` 三级归属，点击可反向跳转。

#### 2.3 生命周期与归档
*   **进行中**：配套活动正常可用，成员可在社团端或赛事端查阅。
*   **赛事收官后**：配套花絮活动在 `CC-5` 与 `CH-5` 均自动进入“归档只读”状态，展示“本赛事已收官，内容已归档”提示，内容永久保留可查阅分享。

## 三、广告 Phase B 搭载逻辑

### 1. 广告位 ID 与 页面映射 (Mapping)

| AdID | 类型 | 场景/路由页面映射 | 触发规则 |
|---|---|---|---|
| AD-B1 | 弹窗 | **AD-B1** 邀请参赛弹窗 | 访问邀请函链接进入小程序即触发 |
| AD-B2 | 弹窗 | **AD-B2** 拉票分享弹窗 | 访问作品详情链接进入小程序即触发 |
| AD-B3 | 弹窗 | **AD-B3** 花絮分享弹窗 | 访问花絮详情链接进入小程序即触发 |
| AD-B4 | Banner | **CH-1** 赛事详情页 | 页面顶部/腰部固定位展示 |
| AD-B5 | Banner | **CH-1-2** 团队排行榜页 | 页面固定位展示 |
| AD-B6 | Banner | **CH-2-2** 获奖证书预览页 | 页面底部/侧边固定展示 |
| AD-B7 | Banner | **CH-5-1** 赛事花絮详情页 | 页面固定位展示 |
| AD-B8 | Banner | **CH-1-3** 赛事作品列表页 | 列表瀑布流中间/底部嵌入展示 |

### 2. 弹窗频次控制（继承 Phase A）
*   AD-B1～AD-B3 弹窗继承 Phase A 频次控制规则：**同一用户单次赛事周期内弹窗总触发次数 ≤ 3 次（跨广告位合并计数）**。超限后该周期内不再触发弹窗，Banner 广告位正常展示不受影响。

### 3. 品牌包装与团长保护
*   **鸣谢机制**：广告弹窗顶部必须展示文字：`“本活动获 [品牌名] 荣誉支持”`。
*   **面子工程**：在 AD-B1 落地页顶端/弹窗显著位置，必须动态展示 `“由 [团长姓名] 邀请您参与”`。系统需根据分享参数中的 `leaderId` 实时反查并渲染姓名。

## 四、荣誉系统（电子发布 & 公示）

### 1. 自动生成规则
*   **触发条件**：运营在后台点击“锁定成绩”或达到预设截止时间。
*   **生成功效**：系统自动合成为所有排名前 N 位的社团生成电子奖项大图。
*   **图片元素**：
    *   赛事 Logo + 主视觉背景。
    *   官方公章（S 端后台配置上传的 PNG 模板）。
    *   获奖社团及团长姓名。
    *   **商业权益**：底部固定展示“官方伙伴/[品牌名] 鸣谢”。

## 五、多状态场景定义

| 场景 | 状态描述 | 规则/展示建议 |
|---|---|---|
| 报名待审核 | 团长端 UI | “资料已提交，请等待官方专员审核...（预计1个工作日内）” |
| 报名被驳回 | 团长端 UI | 展示红色醒目状态，并透传后台输入的“驳回原因”。支持重新发起。 |
| 社团未参赛 | 成员端 UI | 屏蔽“我要投稿”按钮，点击弹出提示：“您的社团暂未参赛，请向团长反馈。” |
| 赛事已结束 | 排行榜/详情 | 停止计票，锁定排名。入口文案由“进行中”变为“已收官”。 |
| 花絮已归档 | 花絮板块 (CH-5/CH-5-1) | 隐藏“上传”入口；展示“本赛事已收官，花絮内容已归档”说明；内容可正常浏览与分享。 |

## 六、运营备注

> **[备注] 广告内容品类约束**：广告内容高调性品类（康养、正规金融、家电、文旅）为运营管理要求，不体现为产品功能约束。由运营人员在后台广告配置层进行人工管控，本期**无需**增加技术侧品类校验功能。
