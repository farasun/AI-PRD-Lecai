# Club Upgrade v1.2 — 社团能力升级

> **迭代主题**：社团能力升级——社团赛事 + 认证工具套件
> **文档阶段**：Step 1 产出（chatA 产品功能描述）
> **核心原则**：不搭建重量级独立模块；社团赛事依托现有社团/赛事框架扩展；认证工具最小化新增，优先复用现有机制
> **关联文档**：`PageList.md`、`flow-framework-v1.2.md`

---

## 一、迭代概述

v1.2 以卡哇伊合作为场景，对社团模块做两个方向的能力升级：

| 模块 | 核心内容 | 设计原则 |
|---|---|---|
| **模块一：社团赛事** | 赛事与社团绑定机制；社团主页展示关联赛事；报名赛事须先入团 | 在现有 CC-4 和 CH-1 上扩展，不新建重量级页面 |
| **模块二：认证工具套件** | 培训资料挂载、认证作品标记、成员认证证书（三个轻量工具）| 复用现有 CC-7 和赛事证书机制，新增页面不超过 3 个 |

---

## 二、模块一：社团赛事

### 2.1 功能定位

社团赛事 = **社团与赛事的绑定关系**，以及由此产生的两个新能力：

1. **社团侧**：社团主页展示关联赛事，让社团成员在社团内发现并进入赛事
2. **赛事侧**：赛事详情展示关联社团信息，报名时前置「入团」检查，引导未入团用户先加入社团

> **与现有模块的关系**：社团赛事不是新模块，是 CC-4（社团主页）和 CH-1（赛事详情）之间新增的关联关系，以及相应的页面改造。

### 2.2 关联模型

**Phase 1 约束**：一赛配一社团（简化），由平台运营后台手动配置绑定关系。Big B（卡哇伊）自助配置不在本版范围。

**Phase 2 可扩展**：一赛配多社团（不在本版范围）。

---

### 2.3 CC-4 社团主页·新增「关联赛事」区块

#### A. 改造说明

当社团有关联赛事时，CC-4 社团主页新增一个「关联赛事」横向滚动卡片区块，展示在活动列表区块的上方（优先级高于普通活动）。

#### B. 区块模块构成

- [Text: Sub-Heading]「关联赛事」区块标题
- 赛事卡片列表（横向滚动）：
  - [Image: Background] 赛事封面图
  - [Text: Body] 赛事名称（如：「2025卡哇伊电吹管公开赛」）
  - [Text: Caption] 赛事状态标签：报名中 / 进行中 / 已结束
  - [Text: Caption] 参赛截止日期（如：「截止 3月31日」）
  - 整张卡片可点击 → 跳转 CH-1 赛事详情页

#### C. 多状态

**[State: Normal] 报名中**
- 卡片正常展示，状态标签显示「报名中」，绿色高亮

**[State: Info] 进行中**
- 状态标签显示「进行中」，蓝色

**[State: Disabled] 已结束**
- 卡片降色展示（opacity），状态标签显示「已结束」
- 仍可点击进入 CH-1，但 CH-1 报名按钮不可用

**[State: Empty] 无关联赛事**
- 不展示「关联赛事」区块，不显示占位符；页面回退为 CC-4 原有布局

#### D. 体验侧重

赛事区块定位是「社团的重要活动之一」，视觉权重略高于普通社团活动，但不喧宾夺主。卡片横向滚动，最多展示 2 张（更多需进入赛事列表）。

### 2.4 CH-1 赛事详情·关联社团展示 + 报名前置入团

#### A. 关联社团信息区块（新增）

当赛事有关联社团时，CH-1 赛事详情页新增一个「主办社团」信息条，展示在赛事标题下方、报名按钮上方。

**区块构成**：
- [Image: Avatar] 社团头像（小，24px）
- [Text: Caption]「本赛事由」+ [Text: Caption Bold] 社团名称 + [Text: Caption]「专属社团主办」
- [Action: Link]「查看社团」→ 跳转 CC-4 社团主页

#### B. 报名前置入团（弹层拦截）

用户点击「参赛报名」时，系统检查是否已加入关联社团：

**已加入** → 直接进入报名流程（无拦截）

**未加入** → 触发引导弹层，内容：
- [Text: Heading]「参与本赛事，需先加入专属社团」
- [Image: Avatar] 社团头像 + [Text: Body] 社团名称
- [Text: Caption] 社团简介（1 行）
- [Action: Primary]「立即加入」→ 执行入团逻辑 → 成功 → Toast「已加入[社团名]！」→ 自动完成报名
- [Action: Ghost]「暂不参赛」→ 关闭弹层

#### C. 多状态（CH-1 报名入口处）

| 状态 | 用户条件 | 行为 |
|---|---|---|
| [State: Normal] 可直接报名 | 已加入关联社团 | 点击「参赛报名」直接进入报名流程 |
| [State: Warning] 入团引导 | 未加入关联社团 | 弹层引导：立即加入 → 自动完成报名 |
| [State: Info] 审核等待 | 申请入团但待审核 | 弹层替换为：「入团申请已提交，审核通过后可参赛」，无「立即加入」按钮 |
| [State: Disabled] 报名截止 | 赛事报名已结束 | 按钮不可点，无入团引导逻辑触发 |

#### D. 体验侧重

用户感知为「一步报名」——点一次按钮，弹层确认加入，自动完成报名，不跳出赛事页面。后台执行「先入团再报名」双步骤，但对用户透明。

## 三、模块二：认证工具套件

认证工具套件 = 三个长在社团下的轻量工具，服务于「卡哇伊师训认证」场景。三个工具无独立模块入口，均依附于社团活动（CC-5/CC-6/CC-7）。

---

### 3.1 工具1：培训资料展示

**定位**：课程资料目录 + 外链跳转，平台不承担视频存储与播放。

#### CC-5 社团活动详情·新增「课程资料」区块

**改造说明**：当活动标记为「培训类活动」时，CC-5 活动详情页新增「课程资料」区块。

**区块模块构成**：
- [Text: Sub-Heading]「课程资料」
- 课程卡片列表（竖向排列）：
  - [Text: Body] 课程名称（如：「第一课·电吹管基础持法」）
  - [Text: Caption] 课程简介（1-2 行）
  - [Action: Ghost]「去学习」→ 跳转卡哇伊提供的外部链接

**多状态**：
- **[State: Normal]** 有课程资料：列表正常展示
- **[State: Empty]** 无课程资料（社团主未配置）：区块不展示，不显示占位

**体验侧重**：培训资料是「参考素材」，不追踪进度，不阻断后续提交流程。

---

### 3.2 工具2：认证作品提交

**定位**：直接复用现有 CC-7 社团提交作品表单，唯一改造点：新增「作品用途」标签。

#### CC-7 社团提交作品表单·新增作品用途标签

**改造说明**：CC-7 表单新增「作品用途」单选项：

| 选项 | 说明 |
|---|---|
| 社团展示（默认） | 普通社团作品 |
| 认证提交 | 作品将作为认证评审依据 |

当活动为「认证类活动」时，默认选中「认证提交」。

**「认证提交」选中时的额外提示**：
- [State: Info] 顶部提示：「此作品将作为认证评审的核心依据，请确认视频清晰完整」
- 提交成功 Toast：「认证作品已提交！完成配套问卷后进入评审」

**体验侧重**：CC-7 表单主体不变，认证模式仅体现在标签默认值和提示文案。

---

### 3.3 工具3：成员认证与证书

认证流程 = 提交作品（CC-7）+ 填写问卷（CC-23）→ 卡哇伊评审 → 平台写入结果 → 复用现有证书机制展示证书（CC-6 + 弹层）。

#### CC-23 教学心得问卷（填写）·新增页面

**功能定位**：作品提交的辅助环节，从 CC-5 活动详情进入，≤5 题选择题。

**界面模块构成（轻量）**：

**模块 1：页面头部**
- [Text: Heading]「教学心得回顾」
- [Text: Caption]「共 X 题·辅助认证评审，非考试」

**模块 2：题目区**
- 每题：[Text: Body] 题目 + 4 个 [Container: Card] 卡片选项，点选即确认
- 选中态：[State: Success]

**模块 3：底部操作**
- [Action: Primary]「下一题」→「提交」
- 提交成功 Toast：「心得回顾已完成！」

**多状态**：
- **[State: Normal]** 未完成：正常作答
- **[State: Disabled]** 已完成：顶部提示「已完成」，内容只读不可修改

---

#### 证书：复用现有赛事证书弹层机制

**实现方式**：认证证书不新建独立页面（CC-24 取消）。复用 CH-2-2 赛事作品详情-证书弹层机制，在 CC-6 社团作品详情页触发。

**触发路径**：
```
认证评审通过
  → 系统通知：「您的认证已通过，点击查看证书」
  → 用户进入 CC-6（认证作品的作品详情页）
  → 作品详情页顶部新增「查看认证证书」[Action: Ghost] 入口
  → 点击 → 复用证书弹层展示认证证书
```

**证书弹层内容（复用 CH-2-2 框架，调整字段）**：

| 字段 | 内容 |
|---|---|
| 证书标题 | 「教学认证证书」 |
| 持证人 | 用户姓名（大字） |
| 认证名称 | 「电吹管教学认证（初级）」 |
| 颁发机构 | 「卡哇伊文化艺术 × 乐才成果平台」 |
| 认证日期 | 系统自动填入 |
| 证书编号 | 8 位字母数字 |

**分享**：复用 CH-2-4/CH-2-5 现有赛事作品分享机制。

**CC-3 个人中心**：认证通过的作品在「我的作品」列表展示 [State: Success]「已认证」角标。无需新增独立「我的认证」模块。

---

### 3.4 配置工具：活动权限 + 问卷配置

#### CC-12-1 活动权限设置（新增极简页面）

**功能**：社团主设置活动可见范围，仅两档。

**界面**：
- [Text: Heading]「谁可以看这个活动？」
- 两个大卡片 Radio：「所有人（默认）」/「仅社团成员」
- [Action: Primary]「保存」

**CC-5 权限锁定态**：非成员进入权限为「仅社团成员」的活动时，内容区显示锁定提示，[Action: Primary]「申请加入社团」。

---

#### CC-12-2 活动问卷配置（新增极简页面）

**功能**：社团主决定是否在活动下挂载「教学心得问卷」。题目由平台运营预置，社团主不出题。

**界面**：
- [Text: Heading]「挂载教学心得问卷」
- Toggle 大开关：开启 / 关闭
- 开启后显示 [State: Info]「成员将在活动详情看到问卷入口，共 5 题」
- [Action: Primary]「保存」

## 四、Phase 1 约束

| 功能 | Phase 1 做什么 | Phase 1 不做什么 |
|---|---|---|
| 社团赛事关联 | 运营后台手动配置绑定关系 | Big B 自助配置赛事绑定 |
| 社团赛事数量 | 一社团最多展示 2 个关联赛事 | 多赛事聚合列表页 |
| 培训资料 | 课程目录 + 外链跳转 | 视频播放、进度追踪 |
| 作品提交 | CC-7 加「作品用途」标签，其余不变 | 新建认证专用提交页 |
| 认证证书 | 复用赛事证书弹层（CC-6 触发） | 新建独立证书页（CC-24 取消） |
| 问卷 | ≤5 题选择题，辅助性 | 评分系统、合格线、重考逻辑 |
| 活动权限 | 公开/成员两档 | 白名单指定成员 |
| 积分联动 | 无 | 认证通过触发积分 |
