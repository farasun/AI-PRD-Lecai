# 李越专家审阅意见 · 卡哇伊合作业务 v1.2

> **审阅角色**：银发经济产品专家·李越
> **审阅对象**：v1.2 卡哇伊合作需求输入文档（师训认证 + 社团赛事配套）
> **审阅日期**：2026-03-19

---

## 一、整体判断

方向是对的，"上传作品→平台认证→奖励兑换"的链路骨架和乐才的核心基因高度契合。卡哇伊选择不搭 e-learning 系统的克制也是正确的——银发用户不需要课程管理系统，他们需要的是"我努力了，有人看见了，给我颁个证"。

但需求文档里有**4处需要厘清、2处有设计风险、1处战略级疑问**。直接说结论，逐条拆开。

---

## 二、逐条审阅意见

### 问题 1：「老师」是 C 端用户还是 Small B？⚠️ 必须厘清，影响整个功能架构

需求文档里的"老师"指的是参加卡哇伊师训的**受训老师**（即要考证的学员）。但乐才平台当前体系中，"老师/社团主"是 Small B 角色，具备发起活动、管理社团等权力。

**这两者是同一批人，还是不同群体？**

我的判断是：卡哇伊师训里的"老师"=乐才体系里的**C 端用户**，他们获得认证之后，才有资格升级为 Small B（开班、建社团）。这个"认证 → 开班资格"的跃升链路，是整个合作最核心的设计点。

**行动要求**：Step 1 产品规划时必须明确角色映射——受训老师在认证前是 C 端，认证通过后可申请成为社团主（Small B）。这不是细节，这是架构问题。

---

### 问题 2：电子认证证书是平台的「荣誉高地」，不是工具，必须当产品来设计 ✅ 核心价值点

对于 45-65 岁的老年大学生来说，"拿到一张证书"是有具体社会意义的——他们会打印出来、放进相框、拿给家人看。

当前需求文档把证书写成了"功能项"（生成、展示、溯源）。这是技术视角，不是产品视角。

**我的要求**：
- 证书必须是「可截图、可分享到微信、有平台水印」的高规格页面，而不是 PDF 或纯文字
- 证书页设计要有「颁发单位（卡哇伊 + 乐才）」「认证级别」「日期」「个人姓名（大字）」
- 分享文案不能写"我通过了测评"，要写："**我获得了[xx]级电吹管教学认证资格**"——这是身份宣言，不是过关通知

---

### 问题 3：「轻量化问卷」的风险 ⚠️ 当心把认证做虚

需求说考试形式是"轻量化问卷"结合作品综合测评。我的担忧是：问卷太轻，认证就没有含金量；问卷太重，银发用户就挫败放弃。

对于 50-65 岁用户，"考试"这个词会触发他们的焦虑记忆（高考、工作考核），必须消解。

**建议设计方向**：
- 把"问卷/考试"改名为「教学心得回顾」或「课程学习确认」——同样是填题，但感受完全不同
- 题目形式：选择题为主（3-4 项），控制在 5 题以内；有图片或视频题干作为辅助
- 通过门槛：建议初期对及格线宽松设置（≥60%），认证重心在作品质量，而非问卷分数
- 关键原则：让每个认真参加师训的老师都能通过认证——认证的价值在于「被平台看见」，而不是「被平台筛掉」

---

### 问题 4：「配套视频课资料」的边界必须界定清楚 ⚠️

需求说"支持卡哇伊上传师训相关配套课程资料（线上视频课），仅做信息展示"。我要问：**这个"展示"是跳转到卡哇伊自己的外链，还是在乐才平台内播放？**

这个边界不界定，Step 3 写 PRD 的时候会卡死。

**我的建议**：
- Phase 1 最轻方案：乐才平台展示课程目录 + 简介卡片，点击后跳转卡哇伊提供的外部播放链接（微信公众号/小程序/优酷均可）
- 乐才平台不承担视频存储、播放、进度追踪的任何责任
- 这样既满足"学习素材引导"的需求，又避免把乐才做成 e-learning 系统

---

## 三、两处设计风险

### 风险 1：「先加入社团，再报名赛事」的操作路径对新用户太重

需求文档附加的赛事需求说：报名分两步——第一步加入专属社团，第二步报名赛事。

这个设计的战略意图我理解（社团是最小活跃单元，赛事必须以社团为纽带），但对于**一个第一次听说乐才平台的卡哇伊学员**来说，这是两次陌生操作，会有明显的流失风险。

**强制要求**：
- 赛事报名页必须有「未加入社团」的状态处理——不是报错，而是**引导式拦截**（"参与本次赛事需先加入[XX社团]，点击一键加入"）
- 加入社团的动作必须在赛事页内完成，不能跳转到社团页面再跳回来
- 设计目标：让用户感知到的是"一步报名"，后台逻辑是"先入团再报名"

---

### 风险 2：分成模式的时机 ⚠️ 战略级疑问

需求文档提到"通过考试相关服务与平台达成合作，探索分成模式实现共赢"。我需要直接说：**Phase 1 试点期，不建议把认证收费或分成写进功能需求。**

原因：
1. 当前乐才平台还在试点期，认证服务的价值还没有被用户验证
2. 收费或分成会提高卡哇伊的预期，也会让平台侧产生"做好了才能要钱"的压力
3. Phase 1 的核心是跑通"上传→认证→证书展示"链路，验证用户的接受度

**建议**：分成模式作为 Phase 2 商议事项写进合作备忘录，当前 v1.2 PRD 不包含任何计费或收益分配逻辑。

## 四、架构建议（S2B2C 视角重组）

根据以上审阅，我重新梳理 v1.2 的角色与流程架构：

### 角色定义

| 角色 | 对应乐才体系 | 在 v1.2 中的行为 |
|---|---|---|
| 乐才平台（S） | 规则制定者 | 提供认证工具、证书生成、赛事框架 |
| 卡哇伊（Big B） | 合作方 / 课程提供者 | 提供课程资料、设计证书样式、制定打分标准 |
| 受训老师（C → Small B） | 初始为 C 端学员 | 看课 → 提交作品 → 完成问卷 → 获得认证 → 有资格创建社团 |
| 赛事参与者（C） | C 端用户 | 加入专属社团 → 报名赛事 → 提交参赛作品 |

### 核心流程（三条主线）

**主线 A：师训认证流程（v1.2 重心）**
```
查看课程资料（轻量展示）
  → 提交培训作品（电吹管演奏视频）
  → 完成「教学心得回顾」（5题以内）
  → 卡哇伊后台评分 + 平台工具出结果
  → 认证通过 → 证书生成 → 可分享/展示
  → [可选] 申请创建社团，成为社团主
```

**主线 B：社团赛事参与流程（配套）**
```
发现赛事 → [未加入社团] 引导式拦截
  → 一键加入专属社团
  → 完成赛事报名
  → 在活动页提交参赛作品
  → 作品关联专属社团
```

**主线 C：证书展示与身份建立（荣誉高地）**
```
认证通过 → 证书页（含姓名/级别/颁发机构/日期）
  → 分享到微信（身份宣言文案）
  → 个人主页展示认证勋章
```

## 五、v1.2 功能范围建议

基于以上分析，建议 v1.2 纳入以下页面，等待 Step 1 产品规划确认：

| 模块 | 建议页面 | 说明 |
|---|---|---|
| 课程资料 | 课程列表页 + 课程详情（外链跳转） | 轻量展示，不做播放器 |
| 作品提交 | 认证作品提交页（区别于社团作品） | 关联认证项目 |
| 认证测评 | 「教学心得回顾」问卷页 | ≤5题，选择题为主 |
| 认证结果 | 认证通过页 + 证书展示页 | 可分享，高规格设计 |
| 赛事配套 | 赛事报名页（含引导式加入社团） | 一体化操作体验 |
| 个人中心 | 我的认证/证书 | 挂载于现有 CC-3 或新增页面 |

CH- 模块（赛事）的具体页面编号，请在 Step 1 时对齐 `foundation/Main_PageList.md`。

| 🟡 中 | 「教学心得回顾」问卷：题量、题型、通过门槛由卡哇伊提供业务逻辑 | 卡哇伊 |
