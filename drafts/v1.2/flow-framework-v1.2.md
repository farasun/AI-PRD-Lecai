## 一、核心设计决策

### 决策 1：培训内容承载方式

**结论**：不新建培训模块。培训内容以"社团活动"为容器，通过活动权限控制访问范围。

| 培训概念 | 平台实现 |
|---|---|
| 一节课 / 培训单元 | 一个社团活动（CC-5） |
| 培训视频 | 活动内视频作品上传 |
| 学员 | 社团成员（或白名单成员） |
| 报名参加培训 | 加入对应社团 |
| 内容访问控制 | 活动权限设置（CC-12-1） |

**核心逻辑**：加入社团 = 获得课程访问权。社团主将培训活动设为"仅成员可见"，非成员无法查看内容。

---

### 决策 2：认证评审由双轨并行触发

作品提交（CC-7 认证模式）和社团活动问卷填写（CC-23）互相独立、无顺序依赖。两项均完成后，系统进入"评审中"状态，由卡哇伊后台完成评分。

```
作品提交 ──┐
            ├─ 两项完成 → 认证评审中 → 通过/不通过
问卷填写 ──┘
```

### 决策 4：问卷是社团工具，不是平台功能

问卷不设独立入口，不在顶层导航出现。它是社团主赋能工具之一：
- **配置权**：社团主在活动管理（CC-12）中挂载问卷（CC-12-2）
- **填写权**：社团成员从活动详情（CC-5）进入填写（CC-23）
- **场景约束**：问卷必须挂载在某个活动下，不能脱离社团独立存在
- **本次试点**：卡哇伊（Big B）是第一个使用方，通过社团工具体系获得赋能

---

### 决策 3：证书由平台提供模板，卡哇伊确认

证书样式基于乐才现有设计调性（品牌色 #E60012，适老化大字版），平台主动出方案，卡哇伊确认，而非等待卡哇伊提供设计稿。

## 二、角色与权限定义

| 角色 | S2B2C层级 | 在v1.2中的行为 |
|---|---|---|
| 乐才平台 | S | 提供认证工具、证书生成、权限控制框架 |
| 卡哇伊 | Big B | 创建社团、发布培训活动、设置活动权限、评审认证作品 |
| 受训老师（认证前） | C端 | 加入社团 → 看课 → 提交作品 + 填写问卷 → 获得认证 |
| 受训老师（认证后） | C→Small B | 获证后可申请创建社团，成为社团主（沿用现有CC-9流程） |
| 赛事参与者 | C端 | 加入关联社团 → 报名赛事 → 提交参赛作品 |

## 三、主线流程定义

### 主线 A：师训认证（核心主线）

**触发场景**：卡哇伊学员完成师训，申请平台认证

**角色流转**：

```
[卡哇伊-Big B]
  创建社团 → 发布培训活动（CC-12）→ 设置权限为"仅成员可见"（CC-12-1）
  → 上传培训视频至活动

[受训老师-C端]
  加入社团（CC-9现有流程）
  → 查看培训活动（CC-5，成员权限）
  → 并行操作：
    ├─ 在活动下提交演奏视频作品（CC-7，认证模式）
    └─ 填写认证问卷（CC-23，≤5题选择题）
  → 系统确认两项完成 → 状态变为"认证评审中"

[卡哇伊-Big B / 平台工具]
  后台查看提交内容 → 评分 → 标记通过/不通过

[受训老师-C端]
  收到认证结果通知
  → 通过：进入CC-24证书页 → 可分享到微信
  → 不通过：可重新提交（规则待定）
```

**关键约束**：
- 培训活动必须设置为"仅成员可见"，否则视频流量不受控
- 作品提交和问卷填写无顺序要求，但两项均完成才触发评审
- 认证通过后，受训老师可选择申请创建自己的社团（CC-9，现有流程，不强制）

---

### 主线 B：活动权限配置（社团主侧）

**触发场景**：社团主创建培训类活动，需限制访问范围

**操作流程**：

```
[社团主-Small B]
  进入CC-12（社团活动管理）
  → 创建新活动 或 编辑已有活动
  → 进入CC-12-1（活动权限设置）
  → 选择可见范围：
      · 全体公开（默认）
      · 仅社团成员
      · 白名单成员（指定特定成员列表）
  → 保存
  → 活动按权限展示给对应用户
```

**关键约束**：
- 默认可见范围为"全体公开"，不影响现有社团活动逻辑
- "白名单"在Phase 1可简化为"仅社团成员"，白名单功能为预留扩展
- 权限变更立即生效，不需要审核

---

### 主线 C：赛事报名含引导加入社团

**触发场景**：用户发现赛事，但尚未加入关联社团

**操作流程**：

```
[用户-C端]
  发现赛事（CC-2赛事导航 → CH-1赛事详情）
  → 点击"参赛报名"
  → [未加入关联社团] 触发引导弹层：
      "参与本次赛事需先加入[XX社团]"
      → 点击"立即加入" → 完成加入 → 自动回到报名流程
  → [已加入] 直接完成报名
  → 报名成功后：
      进入活动页（CC-5）→ 提交参赛作品（CC-7）
      → 作品自动关联社团信息
```

**关键约束**：
- 加入社团动作在赛事页内完成，不跳转到社团页再跳回
- 用户感知为"一步报名"，后台执行"先入团再报名"
- 若社团设置了审核加入，需考虑"审核中"状态下的报名等待态（待Step 3 PRD细化）
