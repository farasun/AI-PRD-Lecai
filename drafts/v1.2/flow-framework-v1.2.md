# v1.2 流程框架定义

> **迭代主题**：卡哇伊合作·社团轻量工具 + 社团赛事配套
> **文档阶段**：Step 1 产出（宏观流程框架）
> **核心原则**：不搭建重量级认证/培训系统；在现有社团功能基础上做最小化扩展
> **关联文档**：`PageList.md`、`certification-feature-v1.2.md`

---

## 一、核心设计决策

### 决策 1：作品提交是核心，认证是结果

所有功能围绕「上传作品 → 平台认证/奖励兑换」的平台核心逻辑展开。认证不是独立的重量级模块，是作品提交完成后的自然结果。

| 原始业务概念 | 平台实现方式 |
|---|---|
| 师训培训内容 | 社团活动（CC-5）下挂载课程资料区块，外链跳转 |
| 学员提交培训成果 | 直接复用 CC-7 作品提交，新增「认证」标签分类 |
| 认证考核 | 轻量问卷（CC-23，≤5 题）作为辅助，主要依据是作品 |
| 认证证书 | 复用现有赛事证书弹层机制（CH-2-2），挂载于 CC-6 |

### 决策 2：三个工具长在社团下，无独立入口

培训资料、作品提交、认证证书三个工具均附属于社团活动（CC-5/CC-6/CC-7），不在平台主导航新增任何入口。对用户而言，操作路径与使用普通社团功能一致。

### 决策 3：不新建认证证书独立页

证书展示复用赛事模块已有的证书弹层机制（CH-2-2），在 CC-6 社团作品详情页触发。原规划中的 CC-24 独立证书页**取消**。

### 决策 4：问卷是辅助工具，不是考试系统

问卷（CC-23）定名「教学心得回顾」，≤5 题选择题，辅助认证评审，不设合格线、不阻断流程。主要评审依据是提交的作品，问卷是参考。

## 二、角色与权限定义

| 角色 | S2B2C 层级 | 在 v1.2 中的行为 |
|---|---|---|
| 乐才平台（S） | 规则制定者 | 提供轻量工具框架、证书机制、权限控制 |
| 卡哇伊（Big B） | 合作方 | 创建社团、配置培训活动权限、上传课程外链、提供问卷题目、评审认证作品 |
| 受训老师（C 端） | 学员 | 加入社团 → 参考课程资料 → 提交作品 → 填写问卷 → 获得认证 |
| 认证后老师（C→Small B） | 升级路径 | 认证通过后可选申请创建社团，沿用现有 CC-9 流程 |
| 赛事参与者（C 端） | 用户 | 加入关联社团 → 报名赛事 → 提交参赛作品 |

## 三、主线流程定义

### 主线 A：社团轻量工具（师训认证）

**触发场景**：卡哇伊学员完成师训，申请平台认证

```
[卡哇伊-Big B]
  创建社团 → 发布培训活动（CC-12）
  → 设置权限「仅成员可见」（CC-12-1）
  → 在活动下添加课程资料外链
  → 挂载「教学心得问卷」（CC-12-2，可选）

[受训老师-C 端]
  加入社团（CC-9 现有流程）
  → 进入培训活动（CC-5，成员权限）
  → 参考课程资料（CC-5，外链跳转，可选）
  → 提交演奏视频作品（CC-7，选「认证提交」标签）
  → 填写教学心得问卷（CC-23，可选但建议完成）
  → 两项完成 → 等待卡哇伊评审

[卡哇伊-Big B]
  通过自有渠道查看提交内容 → 评审 → 通知平台写入结果

[受训老师-C 端]
  收到认证通过通知
  → 进入 CC-6（认证作品详情）→ 点击「查看认证证书」
  → 证书弹层展示 → 分享到微信
  → [可选] 申请创建社团（CC-9 现有流程）
```

**关键约束**：
- 课程资料是参考素材，不追踪是否观看，不阻断后续流程
- 作品提交是核心步骤，问卷填写是辅助，两者无强制顺序
- 认证证书复用现有机制，不新建页面

---

### 主线 B：活动权限配置（社团主侧）

**触发场景**：社团主发布培训类活动，需限制访问范围

```
[社团主-Small B]
  CC-12 社团活动管理
  → 创建/编辑培训活动
  → 进入 CC-12-1（活动权限设置）
  → 选择：全体公开 / 仅社团成员
  → 保存，立即生效

  [可选] 进入 CC-12-2（问卷配置）
  → Toggle 开启「教学心得问卷」
  → 保存
```

**关键约束**：
- Phase 1 仅两档权限（公开/成员），白名单不上线
- 默认值为「全体公开」，不影响现有社团活动逻辑
- 权限配置和问卷配置均为可选操作，不影响活动发布

---

### 主线 C：赛事报名含引导加入社团

**触发场景**：用户发现赛事，但尚未加入关联社团

```
[用户-C 端]
  CH-1 赛事详情 → 点击「参赛报名」
  → [未加入关联社团] 触发引导弹层：
      「参与本次赛事需先加入[XX社团]」
      → 点击「立即加入」→ 加入成功 → 自动完成报名
  → [已加入] 直接完成报名
  → 报名成功后在活动页提交参赛作品（CC-5 → CC-7）
```

**关键约束**：
- 加入社团动作在赛事页内完成，不跳转到社团页再跳回
- 用户感知为「一步报名」，后台执行「先入团再报名」
- 若社团设置了审核加入，弹层提示「申请已提交，审核通过后可参赛」
