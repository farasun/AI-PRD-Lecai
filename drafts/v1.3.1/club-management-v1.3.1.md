## 一、活动可见性权限规则 (Visibility Controls)

为了满足中老年社团在不同发展阶段的需求，新增三类精细化的活动观看权限设置。

### 1. 全平台公开 (Public)
- **定义**：不设限制，面向全网用户。
- **用户行为**：
  - **游客/未登录**：可在瀑布流中搜到并查看活动内容（作品流）。
  - **非社团成员**：可见完整内容，详情页底部展示“申请加入社团”悬浮按钮。
- **业务价值**：用于社团拉新、官方展示，作为公开流量的“钩子”。

### 2. 仅本社团成员 (Club Members Only)
- **定义**：建立社团内私密的社交空间。
- **用户行为**：
  - **已入团成员**：可见完整内容，可互动。
  - **非成员（已登录/未登录）**：
    - 在列表/搜索中可见活动标题与封面图（模糊或占位符处理）。
    - 点击进入详情页后，展示内容锁定态，提示“本活动仅限团员查看，请先申请入团”。
- **业务价值**：提升入团转化率，保护社团内容隐私。

### 3. 指定成员可见 (Designated Members - 核心新增)
- **定义**：面向特定骨干或小组的私密权限。
- **管理端逻辑**：
  - 团长在 `CC-12-1` 中通过**成员授权列表**（内部技术术语：白名单，即通过勾选 User ID 方式实现）进行授权。
  - 若未指定任何成员，默认仅团长本人可见。
- **用户行为（核心隔离规则）**：
  - **被指定成员**：可见且可操作。
  - **非指定成员（包括其他普通团员）**：**执行彻底隐身逻辑**。
    - 列表页、搜索页均不展示该活动。
    - 试图通过活动 ID 二维码或链接进入时，提示“活动不存在或已过期”，不透传权限不足信息。
- **业务核心**：
  - 避免“群内贫富差距”带来的心理落差（不让大妈知道自己没被选中）。
  - 用于核心管理层会议、小圈子福利发放。

## 二、管理路径“穿梭”规则 (Shortcut Navigation)

解决“管理入口太深”的痛点，将管理逻辑前置到展示页面。

### 1. 活动详情页入口 (CC-5 -> CC-12)
- **触发条件**：当前登录用户 = 该社团的团长（或具备管理权限者）。
- **页面改动**：在 `CC-5` (活动详情) 页面的右上角或悬浮位置，新增“管理活动”操作锚点。
- **动作**：跳转到 `CC-12` (该活动的管理项 Action Sheet)，直接聚焦到当前活动的配置。

### 2. 成员列表页入口 (CC-14 -> CC-16-1)
- **触发条件**：当前登录用户 = 团长。
- **页面改动**：在 `CC-14` (社团成员列表) 顶层导航或操作区，新增“管理成员”按钮。
- **动作**：跳转到 `CC-16-1` (成员管理-全部成员)，支持进行禁言、踢出或审核操作。

## 三、跨模块影响 (Cross-Module Impact)

- **数据模型**：需在活动 (`Activity`) 表中增加 `visibility_level` (可见度枚举) 与 `authorized_member_ids` (**成员授权列表**，技术术语：白名单)。
- **成员管理 (`CC-16-1`)**：作为快捷路径的跳转终点，需在当前页面逻辑中适配来自 `CC-14` 的跳转上下文（如：自动聚焦当前社团）。
- **通用展示模块**：需在列表、搜索及详情判定处注入可见性校验逻辑，以支持对非指定人员的“彻底隐身”。

## 四、Phase 边界与约束 (Scope & Constraints)

### Phase 1 (本次迭代)
- 完成以上三种可见性逻辑的后端判定与前端 UI 适配。
- 完成两处快捷跳转入口的实现。
- 完成 `CC-12-1` 权限设置页的设计。

### Phase 2 (预留/不包含)
- 分时段可见（如活动结束后公开）。
- 权限继承逻辑（如：具有特定勋章的成员自动获得访问权）。
- 自动可见规则（基于活跃度自动进入可见名单）。
