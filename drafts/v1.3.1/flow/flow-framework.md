# v1.3.1 页面流程划分框架 (Flow Framework)

> **迭代定位**：基于权限锁定与管理导航的闭环流转。
> **目标**：为 Step 2 的 Chat A 提供确切的流程线。

---

## 1. 核心流程线 (Key Flows)

本次迭代需重点编排以下三条核心流程：

### 1-A：【活动权限配置流程】 (Permission Setting Flow)
- **起点**：`CC-12` (社团活动管理 - 配置项入口)。
- **路径**：`CC-12` -> `CC-12-1` (权限选择) -> `CC-12-1` (指定成员勾选) -> `保存并返回`。
- **关键节点价值**：完成对活动访问权限的物理隔离设定。
- **输出建议**：由 Chat A 制作 `permission-config-flow-A.mmd`。

### 1-B：【活动管理快捷跳转流程】 (Activity Management Shortcut Flow)
- **起点**：`CC-5` (活动详情)。
- **路径**：`CC-5` (详情页) --"判断团长"--> `点击[管理活动]` --"呼起"--> `CC-12` (配置 Action Sheet)。
- **关键节点价值**：缩短团长从浏览到管理的操作链路，实现“即看即管”。
- **输出建议**：由 Chat A 制作 `activity-shortcut-flow-A.mmd`。

### 1-C：【成员管理快捷跳转流程】 (Member Management Shortcut Flow)
- **起点**：`CC-14` (成员列表)。
- **路径**：`CC-14` (列表) --"判断团长"--> `点击[管理成员]` --"跳转"--> `CC-16-1` (成员管理主门面)。
- **关键节点价值**：提高社团长对成员动态的反应速度，增强权力手感。
- **输出建议**：由 Chat A 制作 `member-shortcut-flow-A.mmd`。
