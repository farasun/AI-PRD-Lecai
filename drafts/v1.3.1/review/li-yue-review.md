# 专家前置审阅 (Step 0) - v1.3.1 团长操作与权限优化

**审阅人**：李越（银发经济产品专家）
**审阅时间**：2026-03-27

## 1. 整体业务价值评估 (S2B2C 视角)
本次 v1.3.1 虽然是个小迭代，但精准切中了**“赋能小 B（团长/班长）”**的核心诉求。在中老年社团、班级运营中，团长最怕的两件事：一是“失去对社团内容的掌控感”，二是“操作太复杂，树状菜单太深找不到管理入口”。
- **活动权限细分**：赋予了团长针对不同场景（对外拉新、对内维稳、核心骨干小圈子）的精细化运营工具，这是一个极具价值的“权力下放”，能极大提升团长的平台粘性和“社交荣誉感”。
- **快捷管理跳转**：非常贴合中老年人“所见即所得”的物理直觉动作。他们习惯了在群里看到什么不对就长按撤回，现在在活动详情和成员列表直接提供管理入口，是极其接地气的优化。

---

## 2. 核心需求剖析与修正意见

### 2.1 活动观看权限设置 (CC-12-1 关联)
**原需求描述**：公开权限、仅团员可见、私密权限（白名单）。

**李越的犀利反馈与优化**：
- **物理还原（拒绝互联网黑话）**：
  - 界面上**绝对不能出现“白名单”这三个字**，中老年人会以为自己犯了事被平台拉黑了，或者觉得这就是个看不懂的技术术语。我们必须翻译成物理世界的词汇。
  - **建议落地 UI 文案**：
    - `公开权限` -> **“全平台公开”**（副标题提示：全网随时可看，适合用于社团招募新成员）。
    - `仅团员可见` -> **“仅本社团成员”**（副标题提示：像微信群消息一样，外部人搜不到、看不了）。
    - `私密（白名单）` -> **“指定成员可见”**（副标题提示：仅打勾选中的内部骨干能看，其他人完全不知道有这个活动）。
- **利益与荣誉引擎**：“指定成员可见”实际上是给团长赋予了发“小圈子福利”和“开核心会议”的权力，被选中的成员会产生强烈的“骨干优越感”，有效激励了社团成员的活跃积极性。
- **关联设计确认**：我注意到这与 `drafts/v1.2/flow/flow-B.html` 里的前置设计中“谁可以参与本次师训？”的三个维度完全吻合。架构上是自洽的，可以直接复用并补齐针对“观看”的具体规则。

### 2.2 管理操作前置（CC-5 和 CC-14 的快捷跳转）
**原需求描述**：
- CC-5 跳 CC-12 管理；
- CC-14 跳 CC-20 成员管理。

**李越的犀利反馈与优化**：
- **致命逻辑与编号纠偏**：需求提到“支持快速进入 CC-20 方便管理操作”，但对照 `foundation/Main_PageList.md`，**CC-20 是 Web 平台后管！** 团长在 APP 或小程序上，怎么可能突然跳到电脑端大后台去操作？
- **改进方案**：在 APP 端的 CC-14（社团成员列表），如果判断当前用户是本团团长，应该直接跳转到 **`CC-16-1 成员管理-全部成员`**，这才是对应的 APP/小程序端管理页。
- **场景闭环**：该功能方向我极其赞同。这是一种典型的“巡场式管理体验”：班长在线下班级巡视，看到谁不在名单上直接处理。线上同理，看了列表直接点管理，这种逻辑最短、最有效。

## 3. S2B2C 架构流转动作流

- **S端（平台方）**：赋予了灵活度。但需注意“公开活动”的分发权重池，这是一个未来做流量裂变的好口子。
- **B端（团长/小B）**：
  - 掌握了活动曝光的“发牌权”。
  - 拥有了从 C 端展示页（CC-5, CC-14）一键穿梭回管理大本营（CC-12, CC-16-1）的特权。
- **C端（普通成员/外部受众）**：
  - **公开**：受吸引 -> 申请入团的钩子。
  - **仅团员**：安全、私密的老年人兴趣同温层。
  - **私密（指定人员）**：规则上必须强制做到**对未被指定的人完全“隐身”**。包括在活动列表里都不能出现占位符号！切忌搞出“你无权查看此活动”的提示刺激没被选上的大妈，避免引发不必要的群内矛盾。

## 4. 结论与下一步指引

**结论：** v1.3.1 的业务逻辑高度合理，无合规与商业模式冲突，且极大地照顾了老年的隐私考量和权力欲望。但必须纠正术语（消除白名单）和页面编号映射错误（纠正 CC-20 跳转指向）。
