# PT-1 我的积分 (荣誉与资产中心)

## 状态 I：正常展示态 (资产明细)

### 模块：顶部导航
- **返回操作**：点击后返回 **CC-3 主导航-我的**。
- **页面标题**：「我的积分」，居中。

### 模块：核心荣誉区 (v1.1 核心增量)
- **视觉风格**：深黑色背景 (Black Gold)，白色/金色字体。
- **累计天数 (R1 总量)**：展示用户历史执行过签到（R1）动作的总天数（非连续）。
  - **取值逻辑**：`COUNT(DISTINCT date) FROM t_coin_history WHERE source = 'R1'`。
- **荣誉徽章/寄语**：固定文案「每日打卡，积沙成塔」，作为情感回馈。

### 模块：资产概览与兑换
- **余额数字**：当前可用积分总余额，特大字号展示。
- **兑换指引入口 (Flow D)**：
  - 文案：「兑换礼品 >」。
  - 核心交互：点击弹出 **「线下兑换说明弹层」**（见状态 III）。

### 模块：积分明细 (记录列表)
- **排序规则**：按到账时间倒序排列。
- **每条记录字段**：
  - **来源描述**：口语化文案，对应 v1.1 标准术语（见下方枚举）。
  - **变动数值**：格式「+N」，整数展示。
  - **时间戳**：格式「YYYY-MM-DD」。
- **来源说明文案枚举 (v1.1 对齐版)**：

| 业务行为 | 积分明细文案描述 |
|:--|:--|
| R1 | 每日签到奖励 |
| R2 | 上传作品奖励 |
| R3 | 新手发布作品奖励 |
| R8 | 创建社团奖励 |
| R9 | 发布活动奖励 |
| Milestone | 荣誉达标奖励 |

## 状态 II：空记录态
- **展示逻辑**：
  - 累计天数显示为 `0`。
  - 积分余额显示为 `0`。
  - 记录列表区域替换为：空箱图标 + 文案 **「还没有积分记录，去参与互动开启您的积分之旅吧！」**。
  - **注意**：本页不再提供直接跳转 PT-2 的按钮，引导用户返回 CC-3。

---

## 状态 III：线下兑换说明弹层 (Flow D 落地)
- **形态**：屏幕居中 Modal 弹窗 + 深色模糊遮罩。
- **正文文案**：「线上积分商城正在紧张筹备中。目前，您可以凭借积分，向您的 **班长（社团主）** 报名兑换线下活动的专属福利或精美纪念品。」
- **关键强调**：文案中“班长（社团主）”需通过加粗或下划线强调。
- **关闭机制**：点击「我知道了」或点击外部遮罩，关闭弹窗。

---

## 🛠️ 技术约束与加载逻辑
1. **资产同步**：进入页面时，必须强制从服务端重新拉取最新余额与累计天数，确保与 CC-3 首页的操作结果实时同步。
2. **骨架屏渲染**：在记录列表加载中，优先展示 5-8 行骨架屏背景，防止页面高度跳闪影响阅读体验。
3. **数据兜底**：若接口返回超时，余额区域显示为 `--`。
