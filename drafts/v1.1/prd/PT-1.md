# PT-1 我的积分

---

## A. 核心业务流程 (Logic Flow)

用户从 CC-3「我的」积分卡片点击进入本页。页面展示当前积分余额与全部积分获取记录，按时间倒序排列。用户可上拉加载更多历史记录，可点击底部「如何赚积分」查看规则弹层，了解所有积分获取方式后产生主动参与的意图。

---

## B. 界面模块拆解 (Module Breakdown)

### 模块 1：顶部导航

1. **结构与层级 (Structure & Hierarchy)**

   - *元素构成*：
     - **返回按钮**：[Action: Ghost] 图标：`ri-arrow-left-s-line`，点击返回 CC-3
     - **页面标题**：[Text: Heading] 文案：「我的积分」，居中

2. **核心交互 (Interaction)**
   - `[点击返回]` → 返回 CC-3

---

### 模块 2：积分余额展示区

1. **结构与层级 (Structure & Hierarchy)**

   - *布局逻辑*：横向居中，标签在上，数字在下，单位在数字右侧
   - *元素构成*：
     - **区域标签**：[Text: Caption] 文案：「当前积分」
     - **余额数字**：[Text: Heading]（特大字号）展示整数余额
     - **单位**：[Text: Caption] 文案：「积分」，紧跟余额数字右下角
     - **分割线**：区域底部 1px 细线，与下方列表区分隔

2. **核心交互 (Interaction)**
   - 本区域无交互，纯展示

3. **⚡️ 多状态定义 (State Variations)**
   - **有余额态**：展示实际数字
   - **零余额态**：数字展示「0」，文案保持正常
   - **加载中**：数字区域展示骨架屏占位符

---

### 模块 3：积分记录列表

1. **结构与层级 (Structure & Hierarchy)**

   - *布局逻辑*：垂直列表，每条记录为一行，左侧来源说明，右侧积分数额，底部时间戳
   - *每条记录的元素构成*：
     - **来源说明**：[Text: Body] 展示积分来源的口语化描述（见下方枚举）
     - **时间戳**：[Text: Caption] 格式：「MM-DD」，紧跟来源说明下方
     - **积分数额**：[Text: Body] 格式：「+N」，右对齐
   - *列表区底部*：「上拉加载更多」提示文字（[Text: Caption]）；全部加载完成后展示「已显示全部记录」

   **来源说明文案枚举（完整版）：**

   | 来源编号 | 来源说明文案 |
   |:--|:--|
   | R1 | 每日签到 |
   | R2 | 上传作品至「{活动名称}」 |
   | R3 | 首次发布作品（新手奖励） |
   | R4 | 点赞社团作品 |
   | R5 | 评论社团作品 |
   | R6 | 加入社团 |
   | R7 | 首次加入社团（新手奖励） |
   | R8 | 创建社团 |
   | R9 | 发布活动 |
   | R10 | 社团成员达到 {N} 人 |
   | R11 | 社团作品达到 {N} 件 |

2. **核心交互 (Interaction)**
   - `[上拉到底部]` → 加载更多记录（每次加载 20 条）
   - 每条记录行无点击跳转（Phase 1 不设记录详情）

3. **⚡️ 多状态定义 (State Variations)**
   - **有记录态（Happy Path）**：展示完整列表
   - **空记录态（Empty State）**：
     - *图语义*：[Illu: Empty]
     - *情感文案*：[Text: Body] 「还没有积分记录，去签到或上传作品开始累积吧！」
     - *引导按钮*：[Action: Primary] 「去签到」，点击跳转 PT-2
   - **加载更多中**：列表底部展示 [State: Warning] 加载指示符

### 模块 5：「如何赚积分」规则弹层

1. **结构与层级 (Structure & Hierarchy)**

   - *布局逻辑*：底部弹出浮层（Bottom Sheet），覆盖页面下半部分
   - *元素构成*：
     - **弹层标题行**：
       - **标题**：[Text: Body] 文案：「如何赚积分」，左对齐
       - **关闭按钮**：[Action: Ghost] 图标：`ri-close-line`，右对齐，点击关闭弹层
     - **第一层核心规则（始终展示，3条）**：
       - 行1：图标 `ri-calendar-check-line` + [Text: Body] 「每天打卡签到，可获得 N 积分」
       - 行2：图标 `ri-image-line` + [Text: Body] 「上传作品到社团活动，可获得积分」
       - 行3：图标 `ri-team-line` + [Text: Body] 「加入一个社团，可获得 N 积分」
     - **展开更多入口**：[Action: Ghost] 文案：「查看更多获积方式 ∨」，点击展开第二层
     - **第二层更多规则（展开后显示，默认折叠）**：
       - 「评论社团作品，每天最多 N 次，每次获得 N 积分」
       - 「点赞社团作品，每天最多 N 次，每次获得 N 积分」
       - 「首次发布作品，额外获得 N 积分（新手礼包）」
       - 「首次加入社团，额外获得 N 积分（新手礼包）」
       - 「社团主：创建社团、发布活动、社团达到里程碑可获得额外积分」

2. **核心交互 (Interaction)**
   - `[点击「查看更多获积方式」]` → 展开第二层规则，入口文案变更为「收起 ∧」
   - `[点击关闭按钮]` → 弹层收起，返回记录列表
   - `[点击弹层遮罩区域]` → 弹层收起

3. **⚡️ 多状态定义 (State Variations)**
   - **折叠态（默认）**：仅展示三条核心规则
   - **展开态**：展示全部规则，可上下滚动

## C. 体验侧重 (UX Focus)

本页的核心体验目标是让用户感知到「积分在增长，我的努力被记录了」。因此：

1. 记录列表每条必须明确显示来源说明，不能只展示「+N 积分」，用户需要知道这笔积分从哪来的——这是建立"积分=行为奖励"认知的关键。

2. 「如何赚积分」弹层的分层设计是适老化的核心决策：一次只展示 3 条最重要的规则，避免信息过载导致用户关掉弹层。展开更多是给有兴趣深入了解的用户准备的选项，而非默认展示。

3. 空记录态的引导按钮直接指向「去签到」而非回到首页，减少用户的路径跳转步骤——对银龄用户来说，每少一步都是体验的提升。
