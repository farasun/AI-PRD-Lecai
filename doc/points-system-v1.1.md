# 积分体系说明 V1.1（Phase 1 试点期）

> **迭代范围**：赚积分（行为触发）+ 看积分（余额与记录查看）。积分兑换商城不在本迭代范围，Phase 1 采用人工/线下兑换。

---

## 一、赚积分

### 1. 积分获取行为完整列表

#### 1.1 通用行为（社团主 + 普通成员均适用）

| 编号 | 行为 | 触发条件 | 积分机制 | 防刷约束 |
|:--|:--|:--|:--|:--|
| R1 | 每日签到 | 用户每天首次点击签到按钮 | 固定值（配置表） | 每自然日仅首次点击计分 |
| R2 | 上传社团活动作品 | 在社团活动下成功发布作品（CC-7 提交且通过审核） | 由活动关联的积分规则模板决定（见 1.3 节） | 同一作品首次提交计一次，编辑不重复计分 |
| R3 | 首次发布作品（新手奖励） | 平台生涯首次成功发布作品 | 额外奖励积分（叠加 R2 同时发放，一次性） | 全平台生涯仅触发一次 |
| R4 | 点赞社团作品 | 点赞社团内任意作品 | 固定值（配置表） | 每自然日计分上限由配置表定义（初始建议值：5次/天） |
| R5 | 评论社团作品 | 评论社团内任意作品（有效评论） | 固定值（配置表） | 每自然日计分上限由配置表定义；评论内容需达到最低字数（初始建议值：5字）才计分 |
| R6 | 加入社团 | 成功加入某社团（审核通过或直接加入） | 固定值（配置表） | 每加入一个新社团触发一次，重复申请同一社团不重复计分 |
| R7 | 首次加入社团（新手奖励） | 平台生涯首次成功加入社团 | 额外奖励积分（叠加 R6 同时发放，一次性） | 全平台生涯仅触发一次 |

#### 1.2 社团主专属行为

| 编号 | 行为 | 触发条件 | 积分机制 | 防刷约束 |
|:--|:--|:--|:--|:--|
| R8 | 创建社团 | 社团创建申请审核通过 | 固定值（配置表），一次性 | 每个社团创建时仅触发一次 |
| R9 | 发布活动 | 在社团下成功创建活动 | 固定值（配置表） | 同一社团同一自然日内计分次数上限由配置表定义（初始建议值：2次/天） |
| R10 | 社团团员数达标 | 社团累计成员数达到阶梯值 | 阶梯积分（配置表定义各阶梯分值） | 每个阶梯档位仅触发一次；减员后重新达标不重复触发 |
| R11 | 社团作品数达标 | 社团累计发布作品数达到阶梯值 | 阶梯积分（配置表定义各阶梯分值） | 每个阶梯档位仅触发一次 |

#### 1.3 活动积分规则模板说明（R2）

活动的上传作品积分值由**平台运营侧**管理，**不开放给社团主自行设置**。实现逻辑如下：

- 平台预设若干「积分规则模板」：
  - **默认模板**：普通社团活动的标准积分值
  - **重点推广模板**：平台特别推广的主题活动（如与赛事关联的评比活动），积分值高于默认值
- 活动与积分模板的关联由平台后台运营配置
- Phase 1 以代码配置表实现，不设计运营后台管理 UI

### 2. 积分发放规则

- **实时发放**：所有行为触发条件满足后，平台即时增加用户积分余额。
- **需审核行为**：R2（上传作品）在作品审核通过时发放，审核拒绝不发放。
- **首次行为判断**：R3（首次发布作品）、R7（首次加入社团）的首次判断以用户账号维度记录，不可重置。
- **阶梯积分触发**：R10/R11 的阶梯判断实时计算，达标后立即发放并推送通知；减员后重新增长至已达过的阶梯不重复发放。

### 3. 积分到账通知

- **触发时机**：积分成功到账后，在当前操作页面即时展示。
- **通知形式**：全局 Toast，持续 2 秒后自动消失，无点击跳转。
- **文案参考**（口语化，不用互联网黑话）：
  - R1 签到：「今日签到成功，获得 N 积分！」
  - R2 上传作品：「您的作品已入库，获得 N 积分！」
  - R3 新手奖励：「首次上传成功，平台送您 N 积分！」
  - R4 点赞：「点赞成功，+N 积分」
  - R5 评论：「评论成功，+N 积分」
  - R6 加入社团：「成功加入社团，获得 N 积分！」
  - R7 新手奖励：「首次加入社团，平台送您 N 积分！」
  - R8 创建社团：「社团创建成功，获得 N 积分！」
  - R9 发布活动：「活动发布成功，获得 N 积分！」
  - R10 团员达标：「恭喜！您的社团已集结 X 位成员，获得 N 积分！」
  - R11 作品达标：「恭喜！您的社团作品已达 X 件，获得 N 积分！」

## 二、看积分

### 1. 入口设计

- **主入口**：CC-3 主导航-我的，顶部展示积分卡片模块（卡片式，显示当前余额 + 签到入口），点击卡片跳转至 PT-1 我的积分。
- **每日签到入口**：积分卡片内嵌签到按钮，或点击后跳转 PT-2 每日签到。
- **详情页**：PT-1 我的积分。

### 2. PT-1 核心功能

| 功能 | 说明 |
|:--|:--|
| 积分余额展示 | 展示用户当前积分总余额（整数） |
| 积分记录列表 | 按时间倒序展示全部积分增加记录，含来源说明、积分数额（+N）、时间戳（YYYY-MM-DD） |
| 如何赚积分说明 | 可展开弹层，分层展示积分获取规则（见下方分层要求） |

#### 积分来源说明枚举（PT-1 记录列表中的文案）

| 来源编号 | 来源说明文案 |
|:--|:--|
| R1 | 每日签到 |
| R2 | 上传作品至「{活动名称}」 |
| R3 | 首次发布作品（新手奖励） |
| R4 | 点赞社团作品 |
| R5 | 评论社团作品 |
| R6 | 加入社团 |
| R7 | 首次加入社团（新手奖励） |
| R8 | 创建社团 |
| R9 | 发布活动 |
| R10 | 社团成员达到 {N} 人 |
| R11 | 社团作品达到 {N} 件 |

#### 「如何赚积分」弹层的分层展示要求

- **禁止**：密集表格或长段落，中老年用户会产生排斥感
- **第一层（突出显示，3 条核心规则）**：每日签到 / 上传作品 / 加入社团
- **第二层（可展开查看，其余规则）**：点赞、评论、创建社团（仅社团主可见）、阶梯奖励等
- **文案格式**：使用「您可以……获得积分」的正面句式，用图标+一句话口语描述

### 3. PT-2 每日签到模块

| 状态 | 说明 |
|:--|:--|
| 未签到态 | 显示签到按钮 + 今日可得积分数 + 提示文案 |
| 签到动效态 | 点击签到按钮后触发，显示积分飞出动效 + 积分到账 Toast |
| 已签到态 | 签到按钮变为已签到标记，显示今日已获积分数，不可再次点击 |

## 三、Phase 1 约束与后续计划

| 模块 | Phase 1（本迭代） | Phase 2（下一迭代） |
|:--|:--|:--|
| 积分兑换 | 人工/线下，联系社团长换实物 | 上线积分商城，支持自助兑换 |
| 积分消耗记录 | 无消耗记录，仅展示获取记录 | 展示兑换消耗记录 |
| 积分来源 | 平台自发基础行为（R1–R11） | 引入商家赞助任务积分 |
| 积分有效期 | 无有效期限制，长期有效 | 视运营策略决定是否引入有效期 |
| 运营配置管理 | 代码配置表实现，无管理后台 UI | 上线积分规则运营后台配置界面 |
| 参赛赛事积分 | 待设计（不在本迭代范围）| 纳入设计，参赛投稿触发积分 |
