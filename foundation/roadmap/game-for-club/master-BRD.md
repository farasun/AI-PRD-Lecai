# 乐才官方赛 (Mode A) 核心总规 (Master BRD)

## 1. 业务定义与背景 (Vision)
- **业务定义**：由乐才平台发起的全国/区域性大型赛事（如“京津冀银发风采展”）。
- **核心逻辑**：**仅限社团报名**，严禁个人身份参赛。通过“以团为单位”的强社交绑定，驱动用户建社、入社，沉淀高价值的银发社群作品。
- **术语规范**：强制应用 `foundation/design-system/silver-age-glossary.md` 规范。

## 2. 角色职责与链路 (Architecture)
- **S (平台端)**：
    - 发起并配置赛事，设定“报名最低人数门槛”（可配置，建议 5-10 人）。
    - 负责线下评比，并在后台“名次公示配置区”手动录入获奖结果。
- **B (社团团长)**：
    - 代表社团进行“集体报到”（一键报名）。
    - 负责站外动员（通过官方生成的动员海报）。
- **C (团员/成员)**：
    - 在属地社团报名后获得参赛权，选送作品时与社团荣誉强绑定。

## 3. 核心业务规则 (Business Rules)
- **[R1] 准入门槛**：未达标（人数不足）或未报名的社团成员，投稿页不展示该赛事入口。
- **[R2] 归属逻辑**：
    - 单一身份：直接引导加入荣誉征战。
    - 多重身份：用户需在投稿前确认本次代表哪个已报名的社团“出征”。
- **[R3] 结果公示**：
    - 支持“社团风采录”（人气榜）实时喝彩助力。
    - 最终奖项由管理员在后台配置后，推送到前端专用模块。
- **[R4] 修改权限**：作品一旦成功选送并锁定关联，成员不可私自修改社团归属，需申请团长/后台介入。

## 4. 全生命周期迭代分期 (Phasing)
- **Iteration-A (MVP)**：跑通报名、投稿、后台回填、前端公示的核心链路，术语初步替换。
- **Iteration-B (Advanced)**：补强“动员海报”、“荣誉勋章”、成员归属错误撤回、多身份丝滑引导及数据安全锁定功能。
