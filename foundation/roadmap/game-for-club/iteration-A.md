# 乐才官方赛 (Mode A) 初代切片：核心骨架版 (Iteration-A)

## 1. 迭代范围 (Scope)
- **阶段目标**：跑通 S2B2C 的赛事闭环，重点在于“集体报到”门槛和“手动回填”公示权。
- **业务优先级**：骨干流程 > UI 修饰。

## 2. 核心功能特征 (Feature List)
- **[A-1] 后台赛事配置器**：
    - 新增参数：“社团报到人数下限”（默认 5 人，支持后台自定义）。
    - 绑定“乐才银龄术语表”中的特定标签（如“集体报到”）。
- **[A-2] 团长集体报到流**：
    - 仅限团长可见报名按钮，显示文案：“代表本学社集体报到参加”。
- **[A-3] 成员作品选送流**：
    - 投稿界面顶端增加显性文案：“[社团名称] 已报名此赛事，正在代表社团征战”。
    - 若成员处于多个已报名单，弹出简单的选择列表。
- **[A-4] 运营后台配置区**：
    - 运营录入：支持文件导入或批量配置参赛作品对应的获奖等级（一/二/三等奖）。
    - 前端荣誉公示区：基于 `silver-age-glossary.md` 中的“荣誉墙”语境展示获奖社团及个人作品。

## 3. 技术约束与红线 (Quality Gating)
- **禁用功能**：此版本暂不开发“扫码自动入团后投稿”逻辑。
- **强制约束**：作品一旦提交，其“社团属性”暂不提供自助删除/修改入口，减少业务死循环。
