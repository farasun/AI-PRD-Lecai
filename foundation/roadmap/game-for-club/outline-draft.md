# 乐才赛事A模式功能需求BRD（愿景大纲-草案版）

## 1. 项目愿景与坐标
- **路线图节点**：`game-for-club`
- **业务定位**：平台主导的大型标杆赛事，通过“社团参赛”的强约束，驱动用户建社、入社，沉淀高质量社团作品资产。

## 2. 核心业务流程
- **[发布端] 平台后台**：发起并配置 Mode A 赛事（京津冀等大赛）。
- **[准入端] 报名门槛**：**严禁个人报名**。仅限平台已有的、人数满足基础要求（如 >=2人）的社团报名。
- **[操作端] 团长统筹**：由社团团长代表整个社团点击报名。只有团长报名成功，其成员才具备参赛权。
- **[执行端] 成员投稿**：
    - 团员在赛事入口上传作品。
    - **身份归属决策**：若团员身兼多社团，投稿时**必须手动选择**代表哪一个已報名的社团参加。
    - **作品绑定**：作品实时显示社团标识，自动归档至对应社团相册/赛事动态。

## 3. 结果与激励逻辑
- **社团人气榜**：实时统计各社团参赛作品的点赞/投票总数。
- **评比机制**：
    - 默认形式：线下专家评比 + 线上获奖结果公示。
    - 快速/轻量形式：直接以线上“社团人气榜”锁定后的数据作为排位依据。
- **档案沉淀**：赛事结束后，获奖记录与排行榜永久锁定在赛事页面及社团主页。

## 4. 落地红线要求（低成本模式）
- **极简管理**：无需开发复杂的自动核验奖项逻辑，通过“线下评比 + 后台配置获奖名单”方式公示。
- **归属不可变**：作品一旦选定社团提交，后续绑定关系不可更改，以确保护榜单严谨。
- **报名自动化**：系统自动检索团长身份，不符合条件的团员不开放报名入口。
