# 乐才官方荣耀赛（赛事A模式）功能需求BRD
## 文档基础
基于「赛事模式等事宜讨论」会议记录核心内容梳理，聚焦赛事A模式核心功能需求。本版本已根据专家评审意见进行优化，明确了适老化术语及团长荣誉激励机制。

## 一、项目背景
为打造平台标杆性大赛事，强化平台「社团+赛事」核心生态，依托平台自有资源搭建大型赛事场景，以赛事为引流抓手拉拢用户创建社团、加入社团，提升平台社团整体数量与赛事参与度。

## 二、核心模式定义
赛事A模式为**平台主导搭建的大赛事模式**（UI/用户端统一称呼为：**“赛事”**或 **“官方赛事”**），由平台自主策划、搭建并运营核心大赛事，赛事仅面向平台内已创建的社团开放，**不接受个人参赛**，以社团为统一参赛单位，核心目标是借平台赛事流量推动社团的创建与发展。

## 三、核心功能要点（业务层面）
1. **参赛主体严格限定**
    - 赛事仅对平台内**已创建的社团**开放报名通道，全程不开放个人参赛权限；
    - 为保证社团有效性，报名需满足平台基础的社团人数要求（如：最低5人）。
2. **团长报名与官方审核机制**
    - **报名信息录入**：为降低老年团长操作负担，信息录入遵循“极简原则”，仅需填写/确认：
        * **社团名称**（默认加载，支持微调）；
        * **领队寄语**（参赛誓言或风采简介，展示在赛事页）；
        * **联系电话**（便于紧急情况/获奖后的官方联络）。
    - **双阶段报名流转**：
        * **阶段一：提交申请**。团长填写信息并提交，界面进入“官方审核中”状态；
        * **阶段二：官方审核**。平台后台对社团资质、名称及头像进行行政审核（建议 24 小时内完成），过滤违规或不适宜内容；
        * **阶段三：正式准入/拒绝回复**。审核通过后，社团正式获得“官方参赛队”身份，激活成员投稿权限。
    - **参赛邀请函（荣誉下发）**：**仅在审核通过后**，系统自动生成带平台公章的数字化“官方参赛邀请函”，展示团长姓名及社团名称，支持团长转发朋友圈/微信群进行拉票和荣誉展示。
3. **作品与社团强关联展示**
    - 团员提交的参赛作品，需**明确显示所属社团标识**，与个人作品做清晰区分；
    - 社团板块需实现与赛事页面的**反向链接**，可从社团端直接跳转至对应参赛赛事页面；
    - 参赛作品本质归属对应社团，挂在社团的相册/赛事活动板块下。
4. **赛事配套专属社团榜**
    - 赛事模块需增设**社团专属排行榜**，基于作品人气、投票数等数据实时更新，支持社团拉票互动；
    - **防刷票信誉机制**：排行榜后台配套异常数据过滤机制，识别并忽略营销号刷票，确保官方大赛的公正性与品牌信誉。
5. **赛事结果公示功能**
    - 无专家评审的场景下，直接以**社团人气榜**作为赛事核心结果依据，排行榜数据锁定后不可修改。
6. **赛事荣誉载体：电子证书与奖牌**
    - **荣誉大图生成**：系统需根据最终排名，为获奖社团及个人自动生成带“官方印章”的高清电子证书/奖牌大图；
    - **赞助商冠名**：证书底部预留赞助商品牌露出位，实现荣誉与商业的深度粘合。
7. **情感传播纽带：赛事花絮**
    - **配套活动机制**：社团报名审核通过后，系统自动为该社团创建专属"赛事花絮"配套活动。该活动不可删除、不可改名；团长可通过既有活动权限设置（CC-12-1）控制成员上传权限；
    - **聚合展示**：各社团花絮活动内容自动聚合显示于赛事花絮板块（CH-5），适配广告位 B-3/B-7；赛事收官后花絮活动自动归档只读，内容永久保留；
    - **社交扩散**：花絮支持独立分享，作为高频的社交裂变点，配套专属广告位。
8. **外部访问视图（广告落地页载体）**
    - **小程序外部视图**：针对通过分享链接访问的非 App 用户，以小程序视图（App→小程序、小程序→小程序）提供查阅，无独立 H5 页面；
    - **广告火线位**：该视图为弹窗广告的核心载体，需确保 UI 精美且赞助商信息包容性强。
6. **业务闭环与进度反馈要求（提示项）**
    - **进度可视化**：团长在赛事模块应能实时查看“审核中”、“已准入”或“未通过”的状态，避免由于“黑箱感”导致咨询量激增。
    - **双通道结果触达**：
        * **App 系统消息**：审核结果第一时间通过推送通知告知团长；
        * **短信通知**：针对“审核通过”或“审核驳回（含改进建议）”的关键节点下发短信，确保不习惯长驻 App 的老年团长能及时获知并进行下一步操作（如动员领队寄语或修改信息）。

## 四、落地核心要求
1. **赛事搭建主体**：赛事全程由**平台后台自主搭建、运营与管理**，为平台自有大赛事。
2. **术语规范管理**：在运营位、按钮、分享页等涉及用户操作的 UI 界面上，严禁出现“A模式”等技术术语，必须统一使用**“官方锦标赛”**、**“荣耀大赛”**等权威、具象的称谓。
3. **报名规则适配**：平台赛事规则模块需针对本模式配置专属规则，固定「仅社团可报名、团长极简填报、官方人工审核、生成邀请函」的闭环流程。
4. **结果落地形式**：赛事核心评比环节**线下完成**，平台负责线上真实数据的展示（含刷票过滤后的结果）与最终获奖结果的公示发布。
5. **多端视图适配**：必须同步定义"App 内操作视图"与"小程序外部访问视图"（分享落地场景），广告底座需在小程序外部视图中完整落地。
6. **作品管理归属**：参赛作品及花絮内容需与对应社团做永久绑定，在赛事端、社团端均能实现作品的溯源与查看。

## 五、核心业务价值
1. 打造平台自有标杆大赛事，提升平台赛事的品牌影响力与行业认可度；
2. **强化行政权威感**：通过适老化的正式命名与公章邀请函，满足银发群体对“官方名誉”的渴望；
3. **激活团长积极性**：通过领队寄语与专属荣誉，让团长（小 B）在组织中获得更多“面子”和组织动员力；
4. 降低运营风险，通过自动化的刷票过滤与简化的线上公示流程，在保障信誉的同时控制运营成本；
5. 为平台积累优质的社团赛事作品资源，夯实平台「社团+赛事」的核心生态。
