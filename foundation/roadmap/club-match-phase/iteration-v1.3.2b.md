# 迭代切片：v1.3.2b 团长自助办赛与荣誉闭环

## 1. 迭代目标
在 v1.3.2a 验证增长模型后，补全赛事的荣誉机制，通过电子证书与颁奖公告强化留存；实现办赛权下放，并将生命周期节点从“人工控制”升级为“自动化定时控制”。

## 2. 核心业务规则
### 2.1 权限模式升级 (对齐 v1.3.1)
- 新增支持：**全平台公开参赛 (Public Enrollment)**、**指定成员参赛 (Designated List)**。

### 2.2 荣誉与颁奖系统自动化
- **全自动电子证书/荣誉大图**：自动根据排名生成带章证书。
- **颁奖公告中心**：新增独立喜报页，作为结赛后的沉淀场景。

### 2.3 生命周期自动化 (Automated Nodes)
- **定时任务引擎**：支持团长在发赛时预设各节点的时间点（Visible -> Upload -> Vote -> Result -> Archive）。
- **系统自驱动**：节点到点自动流转，不再需要平台人工操作。

### 2.4 广告进阶支持
- **定义来源**：`foundation/roadmap/advisement/iteration-v1a.md`, `iteration-v1b.md`
- **内容支持**：在 b 阶段进阶功能中，延续并扩展广告模块的埋点支持（含 AD-A 系列及 AD-B 系列）。

### 2.5 团长自助办赛体系
- **APP 端创赛向导**：提供极简表单，由团长自主配置名称、时间轴及奖项。
- **后台授权机制**：平台对特定团长开启办赛特权。

## 3. 关联依赖
- 依赖于 a 阶段跑通的互动及排行底层。
